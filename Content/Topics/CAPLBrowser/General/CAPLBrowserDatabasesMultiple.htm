Use of Multiple Databases

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

With large systems it may be sensible to distribute the descriptions of messages and signals as well as environment variables to several partial databases. Also, when operation with several buses it makes sense to describe each system by its own database.

The simultaneous use of multiple databases is supported. The assignment of databases must be made in the Simulation Setup.

You can find an overview of the assigned databases on the File ribbon tab|Overview.

Afterwards, you can use symbolic names in all function blocks and in CAPL for the messages and signals of all databases. To do this, enter the symbolic name in the appropriate input box. You will find a list of all symbolic names in the signal selection dialogs. The list is opened by activating the small buttons located next to the appropriate input boxes. Then you can select the desired symbolic name from this list.

If you are using more than one database, the messages in the databases following the first database are qualified with the database name. However, you only need these qualified names to resolve ambiguities. As long as the symbolic names are unique in all databases, you can skip qualification of symbolic names in all function blocks and when editing CAPL programs.
