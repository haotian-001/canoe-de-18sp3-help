cclValueBeginBulkUpdate

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Function Syntax

int32_t cclValueBeginBulkUpdate(cclValueID valueID);

Description

Begins a bulk update of a complex value of a distributed object. Use this if you need to update several elements of a value with a complex data type (e.g., a struct or an array) to improve the performance. Each call to cclValueBeginBulkUpdate must be followed by a call to cclValueEndBulkUpdate.

This function can not be used for communication object values or call context values.

Parameters

valueID

ID of the (complex) value, returned by cclValueGetID.

Return Values

CCL_SUCCESS

No error

CCL_WRONGSTATE

The function may only be called during a running measurement, in the PreStartHandler of the measurement and in the StopHandler of the measurement.

CCL_INVALIDVALUEID

The valueID parameter does not identify a known value object.

CCL_INVALIDDOMAIN

The valueID parameter does not identify a distributed object value.

CCL_VALUEREADONLY

The valueID parameter identifies a value which is read-only.

Example

cclValueID completeValue, member1, member2;
completeValue = cclValueGetID(cclDomain::CCL_DISTRIBUTED_OBJECTS, "MyNamespace::MyDO.MyComplexMember", 0, cclValueRepresentation::CCL_IMPL);
member1 = cclValueGetID(cclDomain::CCL_DISTRIBUTED_OBJECTS, "MyNamespace::MyDO.MyComplexMember", "member1", cclValueRepresentation::CCL_IMPL);
member2 = cclValueGetID(cclDomain::CCL_DISTRIBUTED_OBJECTS, "MyNamespace::MyDO.MyComplexMember", "member2", cclValueRepresentation::CCL_IMPL);
cclValueBeginBulkUpdate(completeValue);
cclValueSetInteger(member1, 1);
cclValueSetInteger(member2, 4);
cclValueEndBulkUpdate(completeValue);

Availability

The C Library is executed inside the RT Kernel of your CANoe DE product. So the architecture of the C Library (Windows 32-bit, Windows 64-bit or Linux 64-bit) must fit the architecture of the used RT Kernel.
