Add and Extend the Car2x Database

 

Valid for:  CANoe DE

  • To use application messages (e.g. CAM/DENM (ETSI) or BSM/TIM (SAE)), you have to assign a Car2x database to an ATH channel (e.g. ASNV_Configuration.xml).

    This database is a part of the sample configuration of the Option Car2x and is contained in the DataSources folder.

  • If you want to create a new configuration, you should use a template configuration from the Car2x Demos/MoreExamples folder of the Car2x sample configurations and develop it further.

    To do this copy one of the sample configurations to the folder where you want to save your new configuration.

    The Car2x database and an example sender node are already contained in these sample configurations.

    To add further sender nodes to the database or modify existing sender nodes, open the database with the Car2x Network Explorer.

  • If a configuration does not contain a database yet, assign a database to an ATH channel.

    Then open the database with the Car2x Network Explorer to create sender nodes.

  • If you want to extend an existing database and only have the ASN.1 descriptions and no database file for the respective application message, please proceed as described on page Import Application Messages (Car2x, ASN.1).

  • If you want to combine messages from existing databases, you must copy the database file of the desired application message into the database folder of the target file.

    This folder is called DataSources in the Vector sample configurations.

    If the filename of the old and new application message is the same, you do not need to do anything else.

  • Application messages that are no longer required can be removed in the Car2x Network Explorer via the context menu of the respective message.

  • You can add new application messages in the Car2x Network Explorer using the [Add message] button.

    The new message, the BTP port and the coding of the application message must be specified in the following dialog.
