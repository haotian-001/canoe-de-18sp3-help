Scenario-Based Testing

 

Valid for:  CANoe DE

There are different ways to test scenarios:

  1. Start scenario as defined

  2. Start scenario relative

  3. Start scenario on any route

For each option, a scenario must be created first using the Scenario Editor.

Note

The third option allows for the most flexible testing of scenarios on routes and is recommended to use for most testing in both simulated and real test driving.

As it does not guarantee reproducibility, you should consider the other two options for fixed testing.

Option 1 – Start scenario as defined

The option Start scenario as defined ensures that the scenario is started exactly as defined in the scenario file, which is important for reproducibility.

However, this option requires the scenario to be remade for each route, making it most suitable for use cases where reproducibility is the most important or the scenario is fixed to a test route, such as preemption and prioritization use cases at signalized intersections.

This option can be used without any CAPL code by only using the Scenario Manager.

Option 2 – Start scenario relative to current position

The option Start scenario relative to current position involves starting the scenario as defined but mapped to the current position.

However, this option might lead to simulated vehicles being next to actual roads due to curves or small changes in the heading.

Since the mapping is highly dependent on the current position and heading of the vehicle under test, reproducibility is not guaranteed.

This option is suitable for spontaneous testing, where the underlying route is not known or of importance.

This can be done in one-line CAPL code.

To start the scenario, call the function C2xStartScenario with the current GPS position. The example shows the steps contained in a separate function startScenarioOnRoute, that can be called any time e.g., as an event procedure.

void startScenarioRelative()
{
  // start scenario C2xStartScenario("DuT",
@GNSS::Ath1.Latitude,
@GNSS::Ath1.Longitude,
@GNSS::Ath1.Direction);
}

Option 3 – Start scenario on any route

The option Start scenario on any route involves starting the scenario mapped on a loaded route depending on the current position of the vehicle under test. This option uses distances on the design route in the scenario file as references and requires an additional GPS file describing the route. As it is highly dependent on the current positioning of the vehicle under test, it does not guarantee reproducibility. However, it allows for reusability of scenarios and flexible combinations of scenarios and routes during runtime. This option allows for realistic tests with a pre-known route, both for real driving and HIL-testing in combination with with a driving simulator and GNSS-Simulator.

The Scenario Manager is not used, and no mapping of the nodes is required. The loading and starting of the scenario take place only in CAPL.

Description

Screenshot/Example

1.

Create a GPS File for the route

The GPS File can be created in any way e.g., using the scenario editor or a GPS recording. The valid file formats are KML, GPX or NMEA. It is recommended to save the file relative to the CANoe configuration or in folders below it.

—

2.

Check naming in scenario

The names in the scenario play a role in the following step. Check the name of the following:

  • Station representing the real vehicle under test (commonly name DuT)

  • Design route that the above station is located on (e.g. ScenarioDefinitionRoute)

—

3.

Create CAPL Code

It is recommended to create a separate node containing only the specific CAPL code to start scenario on the route, e.g., named ScenarioControl. Add a CAPL program. Here, the workflow of necessary functions is implemented. The example shows the steps contained in a separate function startScenarioOnRoute, that can be called any time e.g., as an event procedure.

void startScenarioOnRoute()
{
C2xLoadScenario("/Scenarios/MyScenario.scn");

C2xLoadRoute("/Routes/MyRouteFile.kml", "MyRoute", 1);

  // set all stations on loaded route

C2xSetStationsOnRoute("ScenarioDefinitionRoute", "MyRoute");

  // shift so that station HV is at GNSS location

C2xSetRouteStartPoint("MyRoute", "DuT", @GNSS::Ath1.Latitude, @GNSS::Ath1.Longitude, @GNSS::Ath1.Direction);

  // start scenario
C2xStartScenario();
}

4.

Start measurement

Start the measurement and trigger the start of the scenario in the Map window.
