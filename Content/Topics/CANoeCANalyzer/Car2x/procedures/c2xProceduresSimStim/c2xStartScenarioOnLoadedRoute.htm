Start Scenario On Loaded Route

 

Valid for:  CANoe DE

Scenario API

The V2X Scenario CAPL API makes it possible to start scenarios on any route. This is particularly helpful if scenarios are to be started on different test routes, both in real and simulated driving tests. It allows a separation between the design of the scenario and its geographical positioning during execution.

In addition to the scenario file, a description of the GPS data of the execution route is required here. The following formats are supported here: NMEA, GPX and KML. The scenario is loaded and started completely in a CAPL program.

Figure 1: Scenario created in the scenario editor and geographical positioning on the route during runtime

Create Scenario

The scenario is created using the Scenario Editor. The scenario is created by default. The design routes in the scenario file do not play a role and the positioning of the stations is only used as a reference for the correct distances on the route. Care should be taken to deliberately name the station that later represents the Ego vehicle and the design route on which this station is located. These names play an important role in the subsequent CAPL workflow.

Step by Step Procedure

  1. Loading the scenario
    The first step is to load the scenario using the CAPL function C2xLoadScenario.

  2. Load route
    After the scenario is loaded, the route is added to the scenario using the CAPL function C2xLoadRoute. Several routes can be loaded as long as they are uniquely named.

  3. Set stations to the loaded route
    After the route has been successfully loaded, the stations are set to the route from the scenario using the C2xSetStationsOnRoute function. The C2xSetStationOnRoute function can also be used to set a single station to a specific route if not all stations are to be set to the same route. To position the station, its StartOffset value is used, which can be found in the scenario editor under Properties.

  4. Setting the starting point on the route
    If the stations are on the desired route, they can still be moved to the correct position (e.g. the GNSS position). The C2xSetRouteStartPoint function offers two options here. One is to move all stations so that a specified station is located at the specified position. The station representing the ego vehicle should be used here.

  5. Optionally move other routes
    If there are other design routes in the scenario, e.g. an intersecting one, a reference point in the scenario should be used here. The reference point can then be used to move another route to the correct position using the C2xMoveRouteRelativeToRefPoint function.

  6. Start scenario
    The last step is to start the scenario with C2xStartScenario.
