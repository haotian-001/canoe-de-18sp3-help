Task Controller (Window)

 

Valid for:  CANoe DE

The Task Controller window (TC Window) provides an operational Task Controller instance that simulates the ISO11783 communication of a real Task Controller without programming. The Task Controller window independently logs on to the bus, sends its status message, responds to requests sent by implements and diagnostic devices, and receives all uploaded device descriptions (DDOPs). From the loaded device descriptions (DDOP), the Task Controller window calculates the process variable directory for each connected implement and displays it clearly. This provides you with a graphical interface with which you can read, change and query all process variables (including those relevant to Section Control).In addition, the Task Controller window offers a convenient way to analyze the time history of individual selected process variables.

Insert the Window [▲ back]

To add a Task Controller window to the Simulation Setup, open the context menu of the channel in the Simulation Setup by clicking on the bus with the right mouse button. Then click on the Insert ISO11783 Task Controller context menu command. This adds a node to the Simulation Setup. Double click on the node to open the Task Controller Window.

The Task Controller window is divided into two areas. On the left side is the navigation area with four buttons, which you use to show the respective views on the right side. The selected button is highlighted in blue.

In addition, there is a button in the lower left corner, which displays the current operating mode (Offline/Online/Passive). The Task Controller window supports the following operating modes:

  • Offline 
    The Task Controller is not connected to the bus.
  • Online 
    The Task Controller is connected to the bus and is active.
  • Passive 
    The Task Controller is in passive mode.

Click on one of the 3 symbols for the operating mode to open a menu in which you can connect the Task Controller to the bus or disconnect it from the bus and switch on passive mode. You can only switch passive mode on or off when the measurement is not running.

Devices [▲ back]

This view displays all uploaded device descriptions. Each device description is displayed in its own tab. Process variables are indicated by a blue rectangle, while properties are indicated by a green rectangle.

There are two ways to collapse the number of items displayed:

  • Use the and buttons in the toolbar at the top to show and hide all process variables and properties.

  • You can also use column filters: When you move the mouse pointer over a table heading, a filter icon appears in the upper right corner.

Each process variable has a Set button and a Get button.

Using the Get button, request the current value of a process variable via the Request Value Command.

Use the Set button to set a process variable to a new value. Setting is done either by the Value Command or by the Set Value and Acknowledge Command - depending on the setting in the toolbar, which you can access via the buttons and . The arrow color of the button will adjust accordingly:

  • blue when using Value Command.
  • green when using Set Value and Acknowledge Command.

If the corresponding process variable is defined as read-only, the arrow color of the button turns gray. However, the button remains active so that you can check how the attachment responds to the change of a process variable read-only.

For each process variable, the Cmd column displays the available Measurement Commands. Further information can be found in the section Measurement Commands.

The toolbar also contains other buttons:

  • With the task is started. The status message is sent with StatusTotalsActive=1.

  • With the task is stopped. The status message is sent with StatusTotalsActive=0.

  • Using all stored device descriptions are deleted.

Process Variables for Section Control (Condensed State) [▲ back]

For process variables that map the Condensed Work State, there is a special graphical representation that shows the state of each section by a separate rectangle:

The colors have the following meaning:

  • Yellow: deactivated/off (0)
  • Green: activated/on (1)
  • Red: error (2)
  • Gray: undefined/not installed (3)

You can change the status of each section by clicking on the corresponding rectangle. The order is ascending with the left mouse button (gray -> yellow -> green -> red -> gray) and descending with the right mouse button (gray -> red -> green -> yellow -> gray).

Measurement Commands [▲ back]

In the device view, each trigger supported by the respective process variable can be activated by a suitable measurement command. The Cmd column displays a separate button for each supported trigger:

Symbol

Function

Time interval

Distance interval

Threshold Min/Max

On Change

Clicking on the button opens an input mask in which you can configure a suitable measurement command. The measurement command is sent via the green icon or the Enter key. The red icon closes the input screen without sending the measurement command.

Activated triggers are represented by colored buttons, inactive ones by gray buttons.

To deactivate a time interval trigger or a distance interval trigger, you can set its value to 0. For all other triggers (Threshold and On Change), the state of the task must be changed to deactivate them. As triggers can be activated regardless of the state of the task, they are also deactivated when the task is started.

Process Data Tables [▲ back]

Sometimes you need to analyze the time history of one or a few process data variables. For this task, the Task Controller Window provides the specialized Process Data Tables view.

The view is divided into two parts:

  • In the upper collapsible part you can see all device descriptions with their process data variables.

  • In the lower area you can create one or more tables and assign individual process variables to these tables via drag and drop.

Scenarios [▲ back]

In the device description, you can easily read or set individual process variables using the Get and Set buttons. If you want to orchestrate several process variables in a precise chronological sequence instead, the device view reaches its limits. We have developed the so-called scenarios for this purpose. E.g. the tabular view allows you to simulate the diagonal crossing of zone boundaries in just a few steps by switching individual sections on or off one after the other using the SetValueCommand. You can also start or stop the task in the scenario.

The view is similar to the process data tables, as it is also divided into two parts here:

  • In the upper, collapsible section, you can see all device descriptions with their process variables.
  • In the lower section, you can create one or more tables and assign individual process variables to these tables using drag and drop. (See section Process Data Tables).

Step by Step Procedure

Procedure: Using the mouse, drag a line with the process variable you want to set in the scenario from the upper part to the lower part until the lower part is colored light green and release the mouse button there.

Then fill in the table:

  1. Specify the delay in the time column. The specified value determines after how many milliseconds or seconds the values in the row should be sent. Values in seconds are specified with a dot as a separator, e.g.: 1.5. Alternatively, you can specify the unit, e.g. 1.5 s, or 50 ms.
  2. Then enter the values for the process variables. You can also leave cells empty. No value will be sent for these accordingly. The sections view is a special case. As long as you do not change a value in it, the cell is considered empty. This means that to deactivate all sections (all sections in yellow), you must first change the value of a section. You can then deactivate all sections.
  3. If you want to start or stop the task in your scenario, you can do this by activating the Task column in the toolbar and then activating or deactivating the task in the desired row using the buttons.

During the measurement, you can execute, pause or end the scenario using the Play, Pause and Stop buttons. You can also edit the scenario during the measurement. The line that is executed next is highlighted in blue.

You can also add or remove rows using the ribbon or context menu of a row.

Configuration [▲ back]

In the Configuration view, you can set parameters that influence the behavior of the Task Controller window. They are divided into the following groups:

Group

Parameter

Properties

Supported TC version and other data transmitted with the Version message.

Network Properties

In this area, you can configure a device name and an address after the J1939 standard. The device name can either be taken from the database of an existing node or configured manually.

If the selected database node is referenced by a simulation node that in turn uses the Task Controller Interaction Layer (TC IL), the TC window connects to the simulation node. This means that the simulated Task Controller can be operated both via the CAPL program of the simulation node and via the user interface of the TC window.

In passive mode, the network properties defined here are not used.

Language

Here you specify whether the language setting is determined by the primary Virtual Terminal. Otherwise, you specify which data is transmitted with the Language Command message. Changing these properties in the active state results in the message being sent immediately.

Passive Mode

Activate and configure passive mode.

Passive Mode [▲ back]

In passive mode, the Task Controller window does not actively participate in the bus communication, but only listens to the messages on the bus and reconstructs from them everything that happens with the real, active Task Controller:

  • uploaded device descriptions
  • content of process variables
  • status of the task

To determine which Task Controller is to be observed, use one of the following procedures:

Procedure

Description

Observe first detected Task Controller

The Task Controller is observed and displayed, that first sends an Address Claimed message.

Observe Task Controller with address

The Task Controller is observed and displayed that has the same address as specified here is observed and displayed.

Observe Task Controller with Function Instance

The Task Controller is observed and displayed that has the same Function Instance in the device name as specified here.

Device descriptions uploaded to the real Task Controller can be analyzed and exported at any time - even after a communication failure or error in the real Task Controller.

This makes the Task Controller window a true mirror of the real Task Controller.
