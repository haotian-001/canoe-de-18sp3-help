MQTT Security - TLS

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Introduction

MQTT with TLS enables encrypted data transmission. When using TLS, the certificates must be imported into and configured in the Vector Security Manager.

The required changes are described below using the sample configuration Basic MQTT Connectivity and the external broker Eclipse Mosquitto.

Note

It is assumed that a functional MQTT configuration, a broker (Eclipse Mosquitto) and the TLS certificates for the broker, client and CA are available.

Vector Security Manager

The certificates are stored in security profiles in the Vector Security Manager so that your CANoe DE product can access them via a vCDL. How to create such a profile is described below, using the build for a profile for TLS 1.2 as an example.

Setup of the Security Profile Structure

  1. In the Vector Security Manager, on the Home page, in the Security Profile section, click Add.
    This creates a profile with the initial structure.
  2. Expand the selected profile with a TLS 1.2 configuration:
  • Click on Add on page Home in the Parameters area.
  • In the Add Parameter dialog that then opens, set the topmost list to TLS 1.2.
  • Customize the name in the Add parameter dialog. This name will be used by your CANoe DE product in the vCDL.
  • Close the dialog with OK.
  1. In the central window, expand all folders with the TLS Configuration button.
  2. Select the third folder that contains the previously defined name under the Value column.
  3. In the Parameters area, click on Add.
  4. Open the first list in the dialog that then appears and select TLS Client.
  5. Confirm the dialog with OK.

Maintain Certificates

  1. If the certificate for the client has not yet been converted into a PFX file, you should do so, as this format makes it easier to import the client certificate with the Vector Security Manager.
  2. To import the certificates into the security profile, open the list Secrets & Keys Confrigurations.
  3. Select the folder X509 Certificates.
  4. In the Parameters area, click Add.
  5. In the dialog for the import, click on the buttons with the three dots.
  6. Enter the path to the CA.crt and to the client certificate *.pfx.
  7. Select the new folder TLS Client.
  8. Click on the Add parameters button.
  9. In the dialog that then appears, the client certificate that you previously stored under Secrets & Keys Configurations should be preselected. If not, customize the selection via the drop-down list.
  10. Confirm the dialog with OK.

The Vector Security Manager profile can now be used with your CANoe DE product.

CANoe DE product

In a functioning CANoe DE product configuration, such as the sample configuration Basic MQTT Connectivity, the security profile must be transferred to the binding and the URI to the broker must be changed to TLS.

  • In the vCDL, the following attributes must be added to the MQTT Binding DO and set. To set the values, see the description of the attributes:
  • SecurityManager::SecurityProfile
  • MQTTBinding::Authorization::AuthParamName
  • In addition, the URI must be customized for an external broker, e.g. Mosquitto. This URI is located under CANoe DE product Options/Extensions /Connectivity/Mode:WAN Settings. In this window, change the URI from TCP to TLS and change the port if necessary.

Broker

The Mosquitto broker used must be switched to TLS; to do this, the mosquitto.conf must contain the following lines:

  • A port must be defined to which the broker listens: listener 8883
  • The path to the Certificate Authority(CA), Certificate Broker and Broker Key must be specified with the following lines:
  • Path to the CA: cafile path/to/your/CA.crt
  • Path to the broker certificate: certfile path/to/your/broker.crt
  • Path to the broker key: keyfile path/to/your/broker.key
  • The authentication used must be defined (with password or without): allow_anonymous true.
