IoT Enabler

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Introduction

The VH4110 IoT Enabler (Manual: IoTEnabler_Manual_EN.pdf) enables the connection of your CANoe DE product to wireless end devices and sensors. Typical radio interfaces from the IoT environment are supported. The IoT Enabler has direct radio interfaces for WLAN and Bluetooth Low Energy. Other radio interfaces, such as ZigBee or Z-Wave, can be connected to the device in the form of commercially available USB sticks.

The IoT Enabler has an open and extensible firmware interface that you can extend with your own functions. These functions can then in turn be called from your CANoe DE product simulation.

First Steps

  1. Detailed information about the network interface and commissioning can be found in the manual of the VH4110 IoT Enabler.
  2. Configuration and control of the IoT Enabler network interface is performed via your CANoe DE product Communication Setup and a corresponding vCDL description.
  3. The sample configuration IoT Enabler shows the use of the network interface based on a basic example. You can also use this sample configuration as a template for your own configurations.

Connection via the Communication Setup

The VH4110 IoT Enabler is configured and controlled via the Communication Setup. The interfaces of the IoT Enabler are described in a vCDL file in the form of distributed objects. These contain functions that can be called by your CANoe DE product Simulation on the IoT Enabler. Furthermore, these interfaces contain events via which value changes of a connected sensor can be notified. This file must be imported in the Data source area.

The objects described in the vCDL file are then available within your CANoe DE product simulation. They can be used in CAPL or C#, for example, to call methods or to react to events. You can add the corresponding CAPL programs or C# programs in the Application Models area. Furthermore, you can configure these objects in the Graphics Window or Data Window, or control them interactively via a panel.

The sample configuration IoT Enabler contains a vCDL description showing all supported methods and events. Corresponding comments explain the functions and their input parameters and output parameters.

Example

The following example shows a section of the vCDL definition of the Bluetooth Low Energy (BLE) interface of the IoT Enabler:

version 1.4;
import module "IoTEnabler";
namespace IoTEnabler
{
  // Import attributes
  using ::IoTEnablerBinding::Attributes;

  // Import datatypes and interfaces
  using ::IoTEnablerBinding;

  // Configuration of the attached IoT Enabler hardware device
  [Binding="IoTEnabler", LogLevel=DEBUG]
  virtualnetwork IoTEnabler;

  // Instantiation of the device interface
  [virtualNetwork=IoTEnabler]
  object DeviceControl : Device;

  // Instantiation of further interfaces as required, for example:
  [virtualNetwork=IoTEnabler]
  object BLEControl : BLE;
}

Example

The BLEControl object can then be controlled from CAPL as follows:

// CAPL

on key 'a'
{
  // Scan for devices
  IoTEnabler::BLEControl.scan.CallAsync();

  // Connect to a certain device
  IoTEnabler::BLEControl.connect.CallAsync("11:22:33:44:55:66");

  // Subscribe to characteristics
  IoTEnabler::BLEControl.subscribe.CallAsync("00000000-1111-2222-3333-444444444444", "Temperature");
}

on value_update IoTEnabler::BLEControl.notification
{
  if (strncmp($this.name, "Temperature", strlen("Temperature")) == 0)
  {
    write("Temperature is: %d", $this.value[0]);
  }
}
