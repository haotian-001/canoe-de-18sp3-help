AUTOSAR E2E Protection

 

Valid for:  CANoe DE

In order to secure the security-relevant data, the AUTOSAR E2E Protocol Specification defines various profiles that can be used to achieve end-to-end (E2EE) protection. The messages to be protected are assigned a message counter and a checksum.

This protection can also be used under J1939 for parameter groups. The following profiles are supported for J1939 in your CANoe DE product:

  • AUTOSAR E2EE Profile 01
  • AUTOSAR E2EE Profile 05

Configuration

In order for your CANoe DE product to recognize that parameter groups use E2E protection, they must meet the following requirements.

The data to be protected is defined in the database (*dbc) by a signal group. There must be a signal for the checksum (CRC) in the signal group.

Analysis in the Trace Window

The Trace window uses the data from the database to check whether the checksum is correct. The Trace window also checks whether the messages have a consecutive counter number.

If an incorrect checksum or a non-consecutive counter number is detected, an error icon is displayed for the parameter group in question and an error notification appears in the J1939 Violation column:

Simulation with the J1939 Interaction Layer (J1939 IL)

The J1939 IL handles the sending of parameter groups that are defined in the Tx list of the associated node in the database. If the parameter group data is configured for AUTOSAR E2EE Protection as described above, the J1939 IL calculates the checksum to be transmitted when sending and increments the message counter after each transmission.

To inject errors into the E2EE Protection, your CANoe DE product provides the CAPL functions ILNodeDisturbChecksum and ILNodeDisturbCounter. You can use these to falsify the checksum and the message counter.

When using these CAPL functions with the J1939 IL, the parameter counterType is not evaluated.
