Tunnel Protocol Decoders

Ribbon|Hardware Tab|Tunnelprotokoll-Decoder

Valid for:  CANoe DE

The Capture Module Protocol (CMP) defines the communication between the communication participants and the network probes. CMP works as a tunnel protocol and reads the communication on the network. Supported network types are CAN, LIN, FlexRay, and Ethernet. CMP packs the protocol frames into Ethernet packets and tunnels these to the configured devices. At the end, they are unpacked again and mapped to the simulated channels in your CANoe DE product.

The configurations for the Capture Module Protocol can be defined in the Tunnel Protocol Decoders Configuration dialog.

On the Tunnel Protocol Decoders and Infrastructure Access Points pages, you can configure the assignment of the packets to the respective channels and interfaces as follows:

Tunnel Protocol Decoders

On the Tunnel Protocol Decoders page, click on the Add Decoder icon to create a new CMP decoder. Several CMP decoders can be created, but only one of them can be active.

You can then add the subordinate elements to the created CMP Decoder. To do so, open the context menu of the CMP Decoder element and click on Add Device. You can also add CMP Streams to the CMP Device via the context menu, as well as Network Interfaces to the CMP Streams. The elements are displayed under the CMP decoder element in a tree view.

The context menu also provides the option to delete, activate, or deactivate a selected element.

Select an element in the tree view to display its settings on the right side of the dialog. In the settings, you can configure the elements as follows:

Element

Description

CMP Decoder

Name
The name of the CMP decoder.

Active
Determines whether the status of the CMP decoder is active (check box activated) or inactive (check box deactivated).

Only one CMP Decoder can be active at a time.

Evaluate 'InSync'-Flag
Determines whether the InSync flag from the CMP frame header is evaluated (check box activated) or ignored (check box deactivated).

Report unknown mappings
Defines whether packets that are mapped to an unknown device are reported (check box activated) or not (check box deactivated).

CMP Device

Name
The name of the CMP device.

Active
Determines whether the status of the CMP device is active (check box activated) or inactive (check box deactivated).

Device Id
Unique ID of the CMP device.

CMP Stream

Name
The name of the CMP stream.

Active
Determines whether the status of the CMP stream is active (check box activated) or inactive (check box deactivated).

Stream Id
Unique ID of the CMP stream.

Stream Type
The stream type can be used to filter the packets by type. There are the options Raw Ethernet and UDP.

If the UDP option is selected, the destination address and destination port must be configured.

Destination Address
Defines the destination for the filtered packets.

Destination Port
Specifies the port of the destination for the packets.

Determines the sub channels for FlexRay. You can choose between: A, B, and AB.

The name of the port via which the packets are forwarded.

The port name must be named according to the C naming conventions so that it can be addressed from CAPL.

Infrastructure Access Points

On the Infrastructure Access Points page, click on the Add Infrastructure Access Point icon to create a new access point.

Select an element in the tree view to display its settings on the right side of the dialog. In the settings, you can configure the elements as follows:

Element

Description

IAP

Name
The name of the Infrastructure Access Point (IAP).

Active
Determines whether the status of the infrastructure access point is active (check box activated) or inactive (check box deactivated).

IP Address
Unique IP address for forwarding the packets.

Different access points must not have the same IP address.

Subnet Mask
In addition to the IP address, the subnet mask is defined here.
