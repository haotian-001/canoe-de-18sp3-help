Configuration: CANopen

Ribbon: File Tab|Options|Bus Systems/Protocols|CANopen

Valid for:  CANoe DE

SDO

SDO Timeout
You can use this input box to enter the maximum wait time in milliseconds that is to be used to wait for responses from the SDO command.

SDO Block Size
You can use this input box to configure the number of segments for the simulation of the SDO block transfer. Valid values are 1...127.

Use Block Transfer with Threshold (bytes)
For SDO downloads from the CANopen window, the block transfer is automatically used from the set size.

Set NMT status to PreOperational before SDO downloads
If several objects of one or more nodes are written via SDO download (e.g. the entire PDO configuration), the CANopen network is set to the PreOperational state via NMT message before the download.

PDO

Enable system variable updates even with same value
If the check box is activated, every system variable in the object dictionary for which the PDO mapping has been configured is updated when a PDO is received.

If the check box is deactivated, an update only takes place if the value differs from the current value of the system variable. This can increase the data throughput of a CANopen simulation.

EDS/DCF

Save DCF compact
The CiA306 specification optionally provides a more compact storage format for DCF files, which you can activate via this check box.

EDS Folders
This button allows you to specify where your CANoe DE product searches for EDS files in order to display them in selection dialogs and assign devices during scanning.

CMT

Only difference to Default Value
If the check box is activated, the Concise data stream only contains data that differs from the default value from the EDS/DCF.

If the check box is deactivated, all objects contained in the object dictionary of the respective slaves are included in the Concise data stream. This may require more memory space in the CMT master and the configuration of the CANopen network at system startup may take longer.

Include Mappable Objects
If the check box is activated, the Concise data stream also contains values for objects that are marked as mappable and normally change during operation.

If the check box is deactivated, these objects are also configured by the CMT master at system startup.

Name Scheme for CANopen System Variables

The naming scheme for the generation of the CANopen system variables is freely configurable. You can use your own characters and combine the available field codes as you like.

Input Box

Available Field Codes

Namespace for Nodes

Node ID, Node Name

Namespace for Structured Objects

Index (hexadecimal), Object Name (Name of the main object of the structured oject)

Variable for Objects

Index (hexadecimal), Sub Index (hexadecimal), Object Name

The generated names must be C identifiers so that you can access the variable in CAPL. If this condition is not fulfilled, you receive a warning notification after the respective input box.

If the naming schema creates identical names for two objects, they are automatically assigned an index.

The Default Scheme button resets the input boxes to their default settings. In earlier CANoe versions the index and subindex were included in the name, with the Compatible with CANoe < 12.0 button this naming scheme can be set.

LSS

LSS Message Delay (ms)
Use this input box to set the delay when sending multiple LSS messages in a sequence.
