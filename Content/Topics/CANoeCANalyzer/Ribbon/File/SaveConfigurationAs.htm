The active configuration can be saved to a new directory and/or with a new name, together with all sub-files such as CAPL files, logging files, etc.

With a shortcut named Vector <product> in the sidebar of the Save As dialog, you can navigate directly to the directory in which the sample configurations are saved.

Note

  • The display of this link can be deactivated within the Options dialog.
  • CAPL programs containing new CAPL functions of the current product version cannot be compiled and executed with older product versions. This effects all new CAPL features, e.g. new functions. If you want to use the configuration for older compiler versions (older product versions) you shall not use the new features. The source code remains using Save asâ¦ function.
  • Older product versions cannot read the file format of the active configuration. If you still wish to edit active configurations with an older program version, you can save the configurations in the following format that is compatible with the version indicated.
  • Older product versions may offer older versions in the Save asâ¦ dialog.
  • Older product versions prior to version 12.0 SP3 do not yet support module description files (*.vmodule). Modules that are integrated in the network nodes of the Simulation Setup are automatically replaced by the filename of the 32-bit DLL. Modules that are specified in databases (DBC) or in CAPL scripts (#pragma library) are not replaced and are therefore not supported in older versions
  • Select the desired version in the File Type list. However, please note that the older the selected format, the more configuration information that will be lost! Of course, the currently loaded configuration itself remains unaffected by this action.
  • With the New Features pages you can easily track the major changes in each version and therefore the major losses while saving configurations as earlier versions.

Note on System State

Starting with version 9.0 certain system parameters are stored independent of the configuration file. These include, for example, the measurement index and the counter for the test execution.

The file for the system parameters is created in the same directory as the CANoe DE product configuration with the same name but with extension STCFG. If this file is present, it is automatically used in the configuration.

If you copy or move a configuration, ensure that the file for the system parameters is also copied.
