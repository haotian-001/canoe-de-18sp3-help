Attributes of User-defined Trigger Conditions

 

Valid for:  CANoe DE  •  CANoe:lite DE

The attributes of a user-defined condition depend on the bus system of the selected message or bus event.

Common attributes

  • Rx = receive
  • Tx = transmit (send)
  • TXRQ = transmission request (send request)

Bus system specific attributes

CAN

  • Data = Data message
  • Remote = Remote message

J1939

  • Source = Source address of the parameter group (PG)
  • Destination = Destination address of the parameter group
  • Priority = Priority of the parameter group

MOST

  • Function block = ID of the function block
  • Function = ID of the function
  • Operation type = Operation type of the message
  • Source = Source address
  • Destination = Destination address
  • Instance ID

Note

Only messages of type normal are supported.

Use a CAPL program or a MOST Filter to configure other types of messages.

AFDX

  • Ethertype (only in dialog for raw frames): Here you can set the Ethernet Type of a message if only messages of a specific Ethernet Type should be triggered.
  • ICMP (only in dialog for raw frames): Only ICMP messages are triggered.
  • Non UDP (only in dialog for raw frames): Only non UDP messages are triggered.
  • Rx: Only received messages are triggered.
  • Tx: Only sent messages are triggered.
  • Error Flags: Only messages with specific error properties are triggered. If no error flag is active only error free messages are triggered. If the trigger should be valid for all messages, Ignore Error Flag must be activated.

A429

  • Minimum Distance (Min. Dist.): It is triggered if the minimum distance between two consecutive messages is below the specified value.
  • Rx: Only received messages are triggered.
  • Tx: Only sent messages are triggered.
  • Error Flags: Only messages with specific error properties are triggered. If no error flag is active only error free messages are triggered. If the trigger should be valid for all messages, Ignore Error Flag must be activated.
