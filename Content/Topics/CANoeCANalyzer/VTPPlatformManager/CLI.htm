Command Line Interface

 

Valid for:  CANoe DE  •  CANoe:lite DE

The command line interface provides the possibility to perform driver or vtp updates and image updates of VTP devices without user interaction. This is especially useful for automated updating of multiple devices with scripts in a headless environment.

Note

Prerequisite is a network or usb connection to the VTP device.

Client Update Tool

The Client Update Tool takes over the function of the Command Line Interface as an independent program. It is located in the installation directory of the Vector Platform Manager. (typically under C:\Program Files (x86)\Vector Platformanager\ClientUpdateTool.exe)

Program Call

ClientUpdateTool.exe [options]

Note

For each operation, it is necessary to specify connection information to the device.

Every command line operation of the Client Update Tool is processed synchronously. Each call is blocking until the operation is finished and an exit code is returned. Progress information and internal steps are printed on the console.

Available CLI Options

Options

Description

Optional/Necessary

-h [--help]

Lists the description of the parameters

Optional

-i [--ipv4] <arg>

IPv4 address of the target device

Necessary if connected via Ethernet

-y [--usbtype] <arg>

USB type of the target device (VN8900=45)

Necessary if connected via Ethernet

-x [--usbindex] <arg>

USB index of the target device

Necessary if connected via Ethernet

-p [--path] <arg>

Path to the driver files, e.g. "C:\Program Files (x86)\Vector Platform Manager\VtpDrivers"

Necessary if connected via Ethernet

-s [--show-available-components]

Lists the available driver components for the driver path driver path and the connected device.

Optional

-c [--components] <arg>

Installs the transferred driver components of a driver path on the connected device. Takes as argument a comma-separated list of driver names, or a path to a text file path to a text file with the driver names listed in lines. Alternatively, "all" can be passed to update all available drivers. update all available drivers.

Optional

-u [--update-image] <arg>

Updates the image of the target device with the transferred image package (*.vpcimg)

Optional

-ri [--recover-image]

Restores the image to the device if a backup image already exists on the device.

Optional

-b [--backup]

Creates an image backup and saves this as a backup image

on the device.

Optional

-di [--download-image] <arg>

Downloads the backup image from the device and saves it to the target path. Requires a file path with the extension *.vpcimg

Optional

To perform a driver update for specific drivers, enter the connection information, a driver path and the components:

ClientUpdateTool.exe -i [ipaddress] -p [driverpath] -c VN1610,Keyman

To find out the available drivers, first use a call with the "-s" option:

ClientUpdateTool.exe -i [ipaddress] -p [driverpath] -s

If a general VTP update is to be carried out, the following arguments are sufficient:

ClientUpdateTool.exe -i [ipaddress] -p [driverpath]

Exit Codes

In addition to these possible exit codes, the output of the client update tool provides additional information in the event of an error.

Exit Code

Description

0

Success.

-1

The VTP Device or the function is not supported.

-2

Error parsing the arguments.

-3

No connection to the VTP Device could be established.

-6

The update failed.

-10

An operation took too long.

-11

A downgrade was denied.

-12

The driver path is invalid.

-15

General error.

-16

Not enough memory available on the VTP Device.

-17

The VTP Device is no longer supported.

-18

An active license prevents the operation. Deactivate all licenses on the VTP Device via the Vector License Client and restart the device before executing it again. For the Vector License Client an automatable command line interface is also available.
