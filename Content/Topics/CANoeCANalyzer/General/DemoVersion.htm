Demo Version

 

Valid for:  CANoe DE

  • The purpose of the demo version is to give you a basic impression of the program.

  • The feature set is nearly the same as in the full version.

  • The demo version supports the same bus systems as the normal version.

Please note the following restrictions

  • Timer

  • The demo version uses a simple Windows timer.

    Therefore, it is not possible to draw conclusions from the realtime behavior of the demo version to the productive behavior in CANoe DE.

  • Test

  • The test report contains the remark "Demo Version" multiple times.

  • The maximum number of test cases is restricted to 10.

  • The maximum runtime length of test modules is restricted to 60s.

  • When you try to exceed these limits the measurement is stopped and it is documented in the report.

  • Simulation Setup

  • The number of nodes in the Simulation Setup is limited to 4.

    You can load a Simulation Setup with more than 4 nodes but you cannot start it.

  • The configurations you set up with the demo version cannot be run with the /run or /pex variant.

  • You may skip the settings of the bus parameters and the message setup (right-clicking the bus icon in the Simulation Setup).

  • Hardware

  • You cannot connect real hardware to the demo version, because the installed demo driver is used.

  • I/O Hardware

  • You cannot access real I/O hardware.

    For demonstration purposes you can select the two PortLink DLLs pl_demo.DLL and da_demo.DLL.

    Existing CANoe configurations can be edited, e.g. you can remove the PortLink DLLs that aren't supported from the configuration.

  • Windows TCP/IP Stack

  • Access to the Windows TCP/IP Stack (e.g. via CAPL) is not possible.

  • Add-ons

  • You can use the demo Version with the LabVIEW add-on.

    Please note restrictions like the maximum number of nodes.

  • You can use the demo Version with the MATLAB add-on.

    Please note restrictions like the maximum number of nodes.

  • You cannot use the demo version together with the SIL Kit framework.

  • Import/Export

  • You cannot read extracts from logging files.

  • You can import a maximum of 1000 messages.
