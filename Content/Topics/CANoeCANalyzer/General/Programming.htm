Programming Options

Modeling / Testing / Analysis

  • CAPL

    Using the CAPL Browser, you can write CAPL programs (Communication Access Programming Language) for stimulating, simulating, testing and diagnostics (available for CANoe DE, CANoe:lite DE).

    Thus the problem of remaining bus subscribers not yet being available for tests arises when developing control units, for example. To map the system environment, the data traffic of all the other network subscribers can be simulated, for example, using CAPL. (available for CANoe DE, CANoe:lite DE)

    You can also use CAPL to write programs for problem-oriented analysis of the data traffic or program a gateway to exchange data between different buses. (available forCANoe DE, CANoe:lite DE)

  • .NET

    You can use .NET programming languages in different areas:

    • programming of simulated network nodes (available for CANoe DE, CANoe:lite DE)

    • programming of test modules (available for CANoe DE, CANoe:lite DE), test cases and test libraries

    • programming application models

    • programming of snippets (available for CANoe DE, CANoe:lite DE)

  • Python

    You can use Python scripts for programming application models.

  • C API

    The C library feature (CCL for short) enables you to write application code in the C programming language and execute it as part of a measurement.

    A simple C API for accessing services is available in your application code. This API enables the sending and receiving of CAN and LIN frames (available for CANoe DE, CANoe:lite DE), provides timer services and enables access to the system variables.

Communication With External Programs

  • COM

    With the help of the COM Interface you can control your CANoe DE product from external programs.

    Besides applications, scripts also come into consideration as external programs.

  • FDX (available for CANoe DE, CANoe:lite DE)

    FDX (Fast Data eXchange) is a protocol that can be used for easy, fast, and realtime data exchange between your CANoe DE product and other systems via an Ethernet connection.

    The protocol gives other systems read/write access to system variables and bus signals .

    In addition, it is possible to send control commands (e.g., for starting and stopping the measurement) or to receive status information via FDX.

What Programming Language Should Be Used?

  • CAPL

    CAPL is suitable for all use cases The strengths of the programming language are in particular the realtime capability and the many predefined types and functions with which one can directly solve domain-specific tasks. However, for tasks outside the domain -â e.g. parsing XML files or statistical calculations â it is easier to use the libraries of other programming languages.

    Therefore, program in CAPL in particular if

    • there are realtime requirements for the configuration

    • the programs are to access frames or PDUs of bus systems used in the vehicle, or are to influence the bus directly (available for CANoe DE, CANoe:lite DE)

    • the functions of the Test Service Library or for vehicle diagnostics are to be used (available for CANoe DE, CANoe:lite DE)

    • the programs are to influence the simulation of ECUs by using the functions of the corresponding modeling libraries (available for CANoe DE, CANoe:lite DE)

  • C# / .NET

    C# / .NET is useful for tests and application models if

    • there are no special realtime requirements for the configuration (note that a .NET program and especially the loading of .NET assemblies can also have an impact on the rest of the simulation)

    • the programs mainly work on the application level or signal (available for CANoe DE, CANoe:lite DE) level

    • execution on a Linux platform is not required

    • the developers already have experience with the programming language

    • .NET libraries are to be used or code is to be reused by means of classes and assemblies

  • Python

    Python is useful for application models if

    • there are no special realtime requirements for the configuration (note that the execution of a Python program may also affect the rest of the simulation)

    • the programs work exclusively on the application layer

    • the developers already have experience with the programming language

    • Python libraries are to be used

  • C / C++

    C or C++ is useful for simulation models if

    • the developers have experience with the programming language

    • the programs mainly work on the application level or signal (available for CANoe DE, CANoe:lite DE) level

    • external C libraries or generated C code are to be connected

In many cases it can also be useful to combine CAPL with other programming languages. For example, you can call .NET functions or functions of a DLL developed with C/C++ from CAPL; in tests, you can also call CAPL functions from .NET in reverse.

Programs in different programming languages can also communicate using system variables or internal members of Distributed Objects.

vTESTstudio offers even more possibilities to combine different programming languages when developing tests.
