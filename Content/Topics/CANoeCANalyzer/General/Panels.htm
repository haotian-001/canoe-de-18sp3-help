Definition: Symbols

System variables, signals, and diagnostic parameters (valid for CANoe DE, CANoe:lite DE) are referred to as symbols. In the panel environment, symbols can be used to display values or initiate actions.

Screenshot

Overview

Panels are graphical elements via which values can be changed and displayed using so-called controls.

In terms of the controls, a distinction is made between controls and display elements, where certain controls can serve as both control and display elements.

The panels are saved in XVP (extended vector panel) format and can then be loaded in a configurations.

You can create panels with the Panel Designer.

Cross Reference

Detailed information regarding the creation of panels can be found in the Panel Designer help.

For the stimulation, you can use ready-made panels that you can use to change values.

.NET panels can also be integrated.

Control Elements

Display Elements

With controls, you can change values of assigned system variables.

If you are working with transmission models, e.g., as a result of integrating an interaction layer, you can also change values of assigned signals.(available for CANoe DE, CANoe:lite DE)

With display elements you can display changes of system variables, signals, system variables, and diagnostic parameters (available for CANoe DE, CANoe:lite DE).

Note (valid for available for CANoe DE, CANoe:lite DE)

  • With the /run variant, only display elements can be created and configured. Combined control and display elements can only be used for display purposes.
  • Supported symbols depend on the program variant.

Adding to the Configuration

Panel Conversion (available for CANoe DE, CANoe:lite DE)

You can also add a panel to a configuration in the Panel Designer when saving or via Project Documents window.

You can use drag-and-drop to add ready-made panels to a configuration by simply dragging a panel to a desktop.
