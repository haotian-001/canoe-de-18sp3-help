Diagnostic Parameters

Ribbon: Test Tab|Conformance Test Configuration

Valid for:  CANoe DE

The diagnostic parameters are used to execute the transport layer test cases of the conformance tests. They define the content of the diagnostic messages and the behavior during the check.

All user data parameters must be entered as space-separated hexadecimal numbers from 00 to FF. Each number defines one byte of user data. A maximum of 4095 bytes are possible.
Example: F2 02 03 04 05 06 07 08 09 0A 0B 0C 0D

Parameter

Value Range

Description

Diagnostic_FunctionalRequest_SID

[0…175], [184..254]

SID for the sent Functional Request.

Diagnostic_FunctionalRequest

[00…FF]*

Payload for the Functional Request.

Diagnostic_SegmentedRequest_SID

[0…175], [184..254]

SID for the sent Segmented Request.

Diagnostic_SegmentedRequest

[00…FF]*

Payload for the Segmented Request.

Diagnostic_SegmentedResponse_SID

[0…175], [184..254]

SID for the sent Segmented Response.

Diagnostic_SegmentedResponse_Req

[00…FF]*

Payload for the request for the Segmented Response.

Diagnostic_SegmentedResponse

[00…FF]*

Payload for the Segmented Response.

Diagnostic_CheckSegmentedResponseLength

[false, true]

Specifies whether the length of the Segmented Response received should be checked in the test.

Diagnostic_CheckSegmentedResponseContent

[false, true]

Specifies whether the content of the Segmented Response received should be checked in the test.
