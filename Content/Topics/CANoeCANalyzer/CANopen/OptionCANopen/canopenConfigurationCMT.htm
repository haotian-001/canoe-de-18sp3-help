CMT

Ribbon: Simulation Tab|CANopen Configuration

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

On the CMT page, you can configure network management services and configuration master services according to CiA 302 for the CANopen network. A device controls the network startup process as a network management master. As a configuration master, the device can also check the presence of required devices and download their configuration.

NMT/CMT Master

The drop-down list contains all nodes whose object dictionary contains the object 0x1F80 required for NMT master services. Selecting a node activates the NMT master functionality and deactivates it for all other nodes in the network.

Symbol

Description

Reads out all relevant objects for the NMT/CMT master configuration of the selected master node.

Writes all relevant objects for the NMT/CMT master configuration to the selected master node.

Filters the list of nodes contained in the CMT configuration to the nodes contained in the CANopen Configuration window.

Changes the slave assignment settings of the node selected in the lower node list (object 0x1F81).

Changes the NMT startup settings of the master node (object 0x1F80).

Node List

The node list provides an overview of the CMT node configuration in the currently selected CMT master node. The NMT Slave and Required columns allow you to easily change the Slave Assignment setting for the selected node. The Concised Data column can be used to set that the CMT downloads configuration data to the slave. If it is deactivated, the CMT checks the existence of required slaves and, if necessary, their version.

Concise Data

In the lower part of the page, the Concise data stream for the selected node is displayed in the node list.

Symbol

Description

Updates the Concise data with the information currently stored in the object dictionary of the node. Depending on the CMT settings in the Options dialog, all the data or only those that differ from the default value or are not mapable.

Displays only the last value of each object in the Concised data stream in order to be able to view the final result of the configuration process. If the function is deactivated, the entire configuration sequence is displayed. Particularly with PDO configuration, objects must be overwritten several times to enable CANopen-compliant reconfiguration. Objects are therefore contained several times in different places in the Concise data stream.

Reads the Concise data for the selected node from the CMT master.

Writes the Concise data for the selected node to the CMT master.
