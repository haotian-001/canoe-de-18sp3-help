PDO/SRDO Connections

Ribbon: Simulation Tab|CANopen Configuration

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Functions

The PDO/SRDO Connections page provides an overview and configuration of the data exchange in a CANopen network and offers the following functions:

  • Read and write the PDO/SRDO configuration of all nodes of the network.
  • Overview which Tx PDO/SRDO of a node are connected to which Rx PDO/SRDO of other nodes of the network including the communication parameter settings of the PDOs/SRDOs and the mapping objects contained in the PDOs.
  • Overview, which Tx mapping objects of a node are transferred via which CAN ID to Rx mapping objects of other nodes of the network.
  • Consistency check of the object mappings including the data types.
  • Overview of the available PDOs/SRDOs.
  • Connecting Tx and Rx PDOs/SRDOs.
  • Create a counterpart for Tx and Rx PDOs/SRDOs.
  • Editing the communication parameters and mapping parameters.
  • Calculation of the SRDO CRC checksums.

Filter of current view
  • Tx-Rx – Only connected PDOs/SRDOs
  • Tx only – Not connected Tx PDOs/SRDOs
  • Rx only – Not connected Rx PDOs/SRDOs
  • All – Unfiltered view

Connect another Rx PDO/SRDO

Add another Rx PDO/SRDO to the selected Tx PDO/SRDO. The Tx CAN ID is then entered for the selected Rx PDO/SRDO.

Disconnect Tx and Rx PDO/SRDO

With the selected Rx PDO/SRDO a free CAN ID is entered and the Valid flag is removed. If the Tx PDO/SRDO is not assigned to another Rx PDO/SRDO, the Valid flag on the Tx side is also removed.

Set Valid Flags for selected PDOs/SRDOs

Sets the Valid flag of the Tx and Rx PDO/SRDO.

Remove Valid Flags for selected PDOs/SRDOs

Removes the Valid flag on the Rx side. If no further Rx PDO/SRDO with the Valid flag set is assigned, the Valid flag on the Tx side is also removed. With SRDOs, the connection is disconnected because the transmission direction is bound to the valid state.

Creates a counterpart for PDOs and SRDOs

For all valid but not connected Tx and Rx PDOs and SRDOS a CANopen simulation node is created which receives respectively sends these PDOs.

PDO/SRDO View

Displays the PDOs or SRDOs. The Valid column shows the state of the connection:

PDO/SRDO is not connected.

The Mapping-Status column shows whether the mapping is correct or not , or whether it was not checked due to missing entries . Details are displayed in the mapping view.

If the currently selected element is an unconnected Rx PDO, a Tx page can be added via the Tx PDO column.

Via the selection list in the column Rx PDO or Rx SRDO another Rx page can be selected.

Details View: Communication Parameter

This view allows to change the transmission type, the inhibit time, event timer (cycle time) and the SYNC start value for PDOs.

Details View: Mapping Parameter

This view displays the current mapping parameters. You can use the Index column to select another object from the object directory for the current mapping entry. If necessary, the start position of the following parameters will be adjusted. Entries can be added or deleted via the context menu. Only entries from the object directory that are marked as mapable in the DCF and have the access type matching the direction of the PDO are available for selection.

Object View: PDO/SRDO

This view shows which mapped object is mapped to which object of another node via which CAN ID. Here, too, there is a detailed view Communication Parameter.

The Object View does not allow any changes, but the current object in the PDO/SRDO view is selected when switching between the views. This allows easy editing.
