Symbol Recordings

 

Valid for:  CANoe DE  •  CANoe:lite DE

In many cases, analyzing the results in the test report is easier if the progression of the values of signals or other symbols is viewed in the corresponding time range. Symbol recordings in the test report itself offer a convenient way to view the progression of changing values of a symbol in the Test Report Viewer.

Symbol recordings are intended for cases in which it is already known at the time of test design that a manageable number of symbols are relevant for a test scenario or could be influenced by a test case. The test designer can start programmatically control the recording of the corresponding symbols in the test implementation.

In the Test Report Viewer, the recordings can be viewed in a chronological context as a graph for each recorded symbol. Similar to the Graphics window in CANoe DE product, but fully integrated and with many features directly in the Test Report Viewer and available for later analysis of the test without additional tools.

The following CAPL functions are used to control symbol recording:

  • testReportStartRecording
  • testReportStopRecording
  • testReportCreateRecordingGroup
  • testReportAddToRecordingGroup
  • testReportStartRecordingGroup
  • testReportStopRecordingGroup
  • testReportStopAllRecordings

Note

Symbol recordings in the test report are not intended for recording very large numbers of symbols or over long periods of time. For these use cases, logging files should be used, which can then be fully analyzed in vSignalyzer.
