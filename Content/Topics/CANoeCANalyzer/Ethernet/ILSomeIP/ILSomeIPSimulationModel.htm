SOME/IP IL: SOME/IP Simulation Models

 

Valid for:  CANoe DE  •  CANoe4SW DE

Services offered or consumed by SOME/IP ECUs and the attributes of these services (IP address, Service Signals, etc.) can be described in a standardized form. The FIBEX and AUTOSAR formats are used for this. The ECUs described in these databases can be simulated using your CANoe DE product as follows

Note

Your CANoe DE product allows the simulation of SOME/IP ECUs even without a database. The SOME/IP interaction layer provides a comprehensive scope of functions for this.

  1. Create a new configuration.

    • Switch to File ribbon tab.
    • Open New page.
    • Select the Ethernet (Application Objects) template an click Create Configuration From Template.

  1. 2. Assign a SOME/IP database to the Ethernet network.

    Select the Databases entry in the System View of the Simulation Setup with the right mouse button, and select the Add… context menu command.

    Then select the desired SOME/IP database.

  2. Add network nodes in the Simulation Setup.

    In the Simulation Setup, select the line symbol with the right mouse button. Select the Insert Network Node context menu command.

  3. Assign network nodes from the database.

    Open the Node Configuration dialog using the Configuration… context menu command of the added network node. The Network Node list on Common page lists all ECUs described in the database.
    Select your desired ECU here.

  4. Assign the SOME/IP interaction layer.

    • Switch to the Components page in the Node Configuration dialog, and then click the [Add…] button.
    • In the file selection dialog, switch to the exec64 directory of your CANoe DE product installation.
    • Now select the SomeIP_IL.vmodule file.

  5. Close the device configuration.

    Close the Node Configuration dialog using the [OK] button. The node name is now displayed in the Simulation Setup.

    You can now use the CAPL functions for SOME/IP.

  6. Specify the sending behavior.

    You can now define how the simulation model is to transmit the notification messages of the events or fields.
    Select the simulation node with the right mouse button, and select the Component Configuration… context menu command. The sending behavior of the simulation model can be configured in the Component Configuration dialog

    Repeat the configuration process for all required nodes. The IP address stored in the database is already assigned to the simulation models.
    After the start of the measurement, the simulation models execute the Service Discovery. The services are offered according to the database description. If the counterparts (either simulated or real) are present, the simulation models are registered for all consumed services. You can find further information on how to set signal values or react to received signals with the simulation models here.

    Note 2: Assign Multiple Simulation Models

    If you need to assign multiple simulation models to your configuration, you only have to execute steps 1 - 4 for all models.
    The assignment of the SOME/IP interaction layer can then be made simultaneously for all simulation models. Select the Bus node for this (it represents the network itself) in the Simulation Setup with the right mouse button, and select the Assign Module to All Network Nodes… context menu command. Then select the SOME/IP interaction layer as described in step 5.
