SOME/IP IL: BOM

 

Valid for:  CANoe DE  •  CANoe4SW DE

The Byte Order Mark at the start of a data flow serves as an identifier for defining the byte order and coding form in UCS/Unicode character strings.

For SOME/IP UTF8 and UTF16 strings, the following possibilities are available for working with BOM.

UTF8

SomeIPGetValueString() returns the string without BOM.

SomeIPSetValueString() adds the BOM if you have not already taken it into consideration so that the message can be encoded in a valid manner.

Note

With SomeIPSetValueString(), a Configuration String of a Service Discovery option without BOM is set.

See also Syntax for Predefined SD Access.

Possible Error Sources

Example

The UTF8 string param is written in the database with a fixed length of 96 bits (12 bytes), "Hello World" is to be set.

SomeIpSetValueString(messageHandle,"param","Hello World"); // Transfer 11 characters

leads to Error 46-0503, because it implicitly

SomeIpSetValueString(messageHandle,"param","ï»¿Hello World"); // Transfer 14 characters

corresponds and this has a length of more than 12 characters.

The valid characters are nevertheless stored in the parameter so that the message can be sent.

If the stored value in the string is now queried with SomeIpGetValueString, the value “Hello Wo” is returned (11 characters + null terminator).

Error Elimination

It is valid to increase the maximum length to 15 bytes or to shorten the string to a maximum length of 8 bytes (+ 3 bytes BOM + 1 byte null terminator).

UTF16

All UTF16 strings are encoded with the BigEndian BOM þÿ.

Note

UTF16 Little Endian is currently not supported.

If the Byte Order Mark should be excluded when testing or if Little Endian Byte Order Mark is desired, you can achieve this with the SomeIpSetData function.
