Firewall Requirements for the Use of Multi CANoe [▲ back]

As the CANoe DE instances communicate with each other via the computers' infrastructure network, the Windows firewall must allow the required Ethernet packets through. For Multi CANoe, the UDP packets must be allowed through to the ports specified in the Options dialog. The default setting for these ports is 2809.

Note

If CANoe DE is operated in distributed mode, you must observe additional firewall settings. Information on this can be found on the page Used TCP/UDP Ports.

Depending on the firewall settings, it is possible that incoming UDP connections of unknown origin are blocked. In this case, it can be helpful to specify the exact IP address of an agent in the Options dialog for the coordinator (Specify agent addresses option). This establishes a two-way connection between the coordinator and agent. The coordinator starts to reach the agents on its own initiative.

It is also possible that the selected connection profile does not allow the corresponding communication. It can therefore also help to change the connection profile of the network adapter under Windows from Public to Private. You can do this with the following PowerShell command, for example:

Set-NetConnectionProfile -InterfaceAlias "Ethernet" -NetworkCategory Private

Multi CANoe and FDX [▲ back]

Multi CANoe uses the FDX protocol via UDP to exchange data between the various CANoe DE instances. If you have activated FDX in addition to Multi CANoe in the Options dialog under XIL API & FDX Protocol, you must ensure the following points:

  • Multi CANoe always uses the FDX protocol via UDP, so you can only select UDP IPv4 or UDP IPv6 as the transport layer under XIL API & FDX Protocol.
    In addition, the IP version must match: If the Multi CANoe cluster uses IPv6, UDP IPv6 must also be set in the under XIL API & FDX Protocol.
  • The port number in the FDX Settings area under XIL API & FDX Protocol must correspond to the FDX port in the Multi CANoe settings.

Licensing [▲ back]

For Multi CANoe, the same CANoe DE version must be installed on all computers (main version, SP version and hotfix numbers must be identical). Otherwise the measurement start will be aborted.

Multi CANoe requires at least

  • at least one CANoe/pro or CANoe/run license for the computer with the CANoe DE instance in the coordinator role
  • one CANoe/pro or CANoe/run license for each option used in one of the configurations
  • one CANoe Agent license for each additional computer with a participating CANoe DE instance. The CANoe Agent license automatically applies to all options available to the Multi CANoe coordinator.
