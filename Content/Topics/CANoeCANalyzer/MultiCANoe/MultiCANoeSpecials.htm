If several CANoe DE instances of the Multi CANoe environment are to be executed in parallel on one computer or if the CANoe DE instances are to run in distributed mode, some special features must be observed. The following information is based on the content of the Create a Multi CANoe Environment page.

CANoe DE Instances on One Computer [▲ back]

You can also use Multi CANoe to run several CANoe DE instances on one computer. This can increase the overall utilization of the system, as a CANoe DE instance does not usually use all processors at the same time.

CANoe DE requires a lot of main memory, especially for large configurations. For this reason, it is important to ensure that the computer is equipped with sufficient main memory.

Network Ports

If several CANoe DE instances are running on the same computer, you must specify different FDX ports for all CANoe DE instances in the Options dialog.

Channel Mapping

If several CANoe DE instances are operated on one computer, they should generally be mapped to different hardware channels.

In the Options dialog, you can use the Use instance name as application channel mapping name option to give the current CANoe DE instance (for agents only) the selected instance name as the application name.

You can make the channel mapping for this application via the Hardware ribbon tab | Channel Mapping (or alternatively via the Vector Hardware Manager).

By default, the instance name CANoe is used for the coordinator. If you want to change the instance name of the coordinator, the CANoe DE instance of the coordinator must be started with the command line parameter CANoe64.exe /m <Instanz-Name>.

CANoe DE in Distributed Mode [▲ back]

If you use CANoe DE instances in distributed mode, in which the simulation is transferred to a network interface, you must pay attention to the following aspects when configuring the Multi CANoe cluster.

The settings required to operate Multi CANoe with CANoe DE instances in distributed mode are explained here using two examples.

Identification of the Runtime Kernel Instances

A Multi CANoe environment consisting of two CANoe DE instances is considered here. The two CANoe DE instances run in distributed mode on separate computers, so that there are two GUI computers and two computers for the runtime kernel.

For the individual CANoe DE instances, you must connect the computer with the GUI and the computer with the runtime kernel. In the example above, there must therefore be an Ethernet connection between PC 1 and PC 2, as well as between PC 3 and PC 4. In addition, all runtime kernels must be connected to each other so that they can communicate with each other via FDX. The recommended setup is therefore to connect all PCs to the same Ethernet LAN, as shown in the picture above.

In the Options dialog, you must always enter the IP address of the computer on which the runtime kernel is running. For the setup above, this would be the IP address of PC 2 and PC 4.

As usual in distributed mode, the network events are received by the network interfaces and sent to the RTK computer with a time stamp. From there, it is forwarded to the GUI computer for display. You can configure the connection between the RTK and GUI computer via the Vector Tool Platform on the CANoe DE Home ribbon tab.

Configure Multiple Runtime Kernel Instances on a VTP Device

It is also possible to run multiple CANoe DE runtime kernel instances on the same VTP device. In the following example, two instances of the CANoe DE GUI are running on a GUI computer and the corresponding runtime kernel instances are running on one VTP device:

To create the runtime kernel instances, you must connect the VTP device on which the instances are to be executed to the GUI computer via Ethernet. Then open the Vector Tool Platform via the Home ribbon tab. There you can create the necessary runtime kernel instances under CANoe/CANalyzer | Execution Environment in the Multi CANoe area. To do this, click [Create] for each instance required and enter a unique name. The names of the runtime kernel instances can be the same as the instance names in the Multi CANoe settings. However, this is not a requirement, but it makes assignment easier.

The RT kernel instances created in this way can only be used for execution with Multi CANoe. However, there is still the standard instance with the name [Standard], which you can continue to use for all CANoe DE setups - both with and without Multi CANoe.

If you have created a runtime kernel instance, it will appear for selection under CANoe/CANalyzer | Distributed Mode so that a connection to this instance can be established there.

You can also pass the runtime kernel instance names in the command line when starting CANoe DE. You can set this option using the /c parameter as follows

.\CANoe64.exe /c Instance1@192.168.0.1

Instance1 corresponds to the instance name of the RT kernel and 192.168.0.1 to the IP address of the network interface.

If the runtime kernels run separately on two separate computers, you do not need to assign runtime kernel instance names. The runtime kernel running by default is used automatically. The instance name can then be omitted from the command line parameter /c, e.g. only /c 192.168.0.1.

Note

The feature to run multiple runtime kernels on one VTP device is limited to the following device: VN8914, VN8917, VT6060, RT-Rack CS246E, RT-Rack AS246E.

In particular, devices of type Unknown are not supported.

Multi CANoe can also be operated in standalone mode. An example setup with two standalone mode instances looks like this:

The following scenarios are supported:

  • All instances (including the coordinator) can be operated in standalone mode.
  • Only individual instances can be operated in standalone mode. The other instances can then run either in non-distributed or distributed mode.
  • If multiple runtime kernel instances are configured on a VTP device, multiple standalone instances can also be run on a VTP device.
