CANoe DE

Diagnostic Window

The Diagnostic window combines several Diagnostic Features within one window. You can set the diagnostic features displayed in the Diagnostic window in the Diagnostic Window Configuration dialog. Currently you can configure the Diagnostic Features ECU Control, Variant Coding, Diagnostic Console and Fault Memory. The number of diagnostic features in the Diagnostic window will be gradually expanded in future Diagnostics with your CANoe DE product versions.

CANoe DE

Diagnostic Parameters Window

In the Diagnostic Parameters Window, you can insert diagnostic response parameters and configure the associated requests. After that you can send requests interactively via mouse click, or cyclically immediately after the measurement starts, and the parameter values contained in the responses will be displayed both in the Diagnostic Parameters Window and in the other analysis windows (the State Tracker, Data Window, and Graphics Window).

CANoe DE

Diagnostic Session Control

The Diagnostic Session Control allows to switch an ECU interactively between its different diagnostic states (changing a session, security access) as well as to control its communication management settings.

CANoe DE

Diagnostic Console

The Diagnostic Console makes it possible to send single diagnostic requests directly to an ECU and receive and analyze the corresponding diagnostic responses.

CANoe DE

Fault Memory Window

The fault memory list of an ECU can be read directly with the Fault Memory Window. You can also delete individual entries in the fault memory list (DTC - Diagnostic Trouble Code).

To read out the fault memory list of an ECU, the respective database must be loaded in a diagnostic description format (CDD, ODX or MDX).

CANoe DE

OBD-II Window

The OBD-II Window makes it possible to execute On-Board Diagnostics (OBD) according to the SAE J1979 standard.

CANoe DE

Basic Diagnostic Editor

Using the Basic Diagnostic Editor, you can describe simple diagnostic services (UDS & KWP) for control units on CAN, LIN, FlexRay and K-Line.

CANoe DE

Diagnostic/ISO TP Configuration Dialog

On the left in this dialog you find a tree view via that you can access supported diagnostic functions. E.g. you can add diagnostic descriptions to one or more networks and modify dedicated parameters.

CANoe DE

Diagnostic Window Configuration

In this dialog you can assign the available Diagnostic Features to one Diagnostic Window or distribute the features to several of these windows.

Currently you can configure the diagnostic features:

  • ECU Control
    The ECU Control is a Diagnostic Feature that you can use within the Diagnostic window. The ECU Control is used to change the diagnostic session for an ECU, to enable the security access in the ECU and to authenticate the tester at the ECU. Security aspects from any configured security sources are taken into account. Furthermore, ECU resets can be executed, DTC setting services and communication services can be sent.
  • Variant Coding
    The Variant Coding is a Diagnostic Feature that you can use within the Diagnostic window. The diagnostic feature Variant Coding is used to read, write and compare variant coding data of an ECU. Security aspects from possibly configured security sources are taken into account.
  • Diagnostic Console
    The Diagnostic Console is a Diagnostic Feature that you can use within the Diagnostic window. In the Diagnostic Console it is possible to directly send individual diagnostic requests of an ECU, display and evaluate the corresponding diagnostic responses.
  • Fault Memory
    The Fault Memory is a Diagnostic Feature that you can use within the Diagnostic window. You can read out the fault memory of an ECU and delete all or individual entries in the fault memory (DTC - Diagnostic Trouble Code).

J1939/ISO11783 [▲ back]

CANoe DE

J1939 DTC Monitor

With the J1939 DTC Monitor in the Measurement Setup you can easily display and request error codes (DTC – Diagnostic Trouble Codes) and diagnostic messages. With the support of Freeze Frames (extended information from fault memory) you can use the J1939 DTC Monitor for emission related or power train diagnostics.

CANoe DE

J1939 Diagnostic Memory Access

With the J1939 diagnostic memory access window you can read and write parts of the memory as well as execute a boot load procedure. For this the diagnostic messages DM14, DM15, DM16 and DM18 are used.

CANoe DE

J1939 OBD-I/M Monitor

The J1939 On-Board Diagnostic Inspection and Maintenance Monitor (short: J1939 OBD-I/M Monitor) allows to configure diagnostic messages of type DM7 and send them to ECUs in the network.

J1587 [▲ back]

CANoe DE

J1587 DC Monitor

The J1587 Diagnostic (DC) Monitor examines transmitted diagnostic messages (PIDs 194…196), displays their contents, and enables requests in order to retrieve diagnostic data.
