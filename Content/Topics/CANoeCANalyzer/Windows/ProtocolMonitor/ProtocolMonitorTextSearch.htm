Text Search

 

Valid for:  CANoe DE

To keep an overview in large networks, the Protocol Monitor offers a text search. You can open the text search via the icon in the global toolbar.

If you enter a search text, only the objects whose properties contain the desired search text will be displayed. If you do not enter a search text, then all objects will be listed.

Columns

Column

Description

Type

Specifies the type of the found object:

  • Network Node
  • Network
  • Endpoint
  • Connection
  • Validation Result
  • Service (SOME/IP)
  • Service Instance (SOME/IP)
  • Service Consumer (SOME/IP)
  • Service Provider (SOME/IP)
  • Subscription (SOME/IP)
  • Eventgroup Subscription (SOME/IP)

Object Name

Contains a description of the identity of the found object.

Matches

Contains the properties because of which the found object is a search hit.

If the search text is empty, all properties of the object are listed.

Search Settings

You can change the search settings using the icons next to the search field.

Setting

Function

Extended Mode

If this setting is activated, then you must enter a search text in the format of the Protocol Monitor Query Language. This also allows more complex search queries to be realized.

If the entered text is incorrect, the input field is highlighted in red and a tooltip provides further information about the input error. As an aid, a list of valid input options is displayed when typing. In the footer of the text search, the search text is displayed that was used to create the currently displayed search hits.

Regardless of the data type, the properties are all interpreted and searched as text.

If this setting is not activated, the entered search text is used directly.

Match Case

If this setting is activated, then only search hits are found where the upper and lower case letters match those in the entered search text.

This setting has no effect in Extended Mode, where case sensitivity is not taken into account.

Match Whole Word

If this setting is activated, then only search hits are found where the entered search text corresponds to a whole word.

This setting has no effect in Extended Mode, where all hits are considered regardless of word boundaries.

Types

With these settings you can reduce the object types to be searched:

  • Communication View
  • Network Node
  • Network
  • Endpoint
  • Connection
  • Validation Overview
  • Validation Result
  • SOME/IP Overview
  • Service
  • Service Instance
  • Service Consumer
  • Service Provider
  • Subscription
  • Eventgroup Subscription
