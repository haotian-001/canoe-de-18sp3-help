Extended Real Time (ERT)

 

Valid for:  CANoe DE

The aim of Extended Real Time (ERT) is to fulfill deteministic requirements during simulation execution. Maintaining latency and low jitter are of great importance here. Therefore, only the simulation parts or functions that are explicitly subject to deterministic requirements are executed in ERT.

ERT is an extension to simulation execution and is therefore focused on specific functions.

ERT enables the execution of CAPL programs and MATLAB simulation models under realtime conditions up to typical cycle times of 100Âµs. ERT also supports several CAN/CAN-FD network interfaces and XCP on CAN. By directly addressing the I/Os of the VT system from ERT, they can also be used under realtime conditions. To configure and use ERT on a VT system, there is more information and detailed instructions on this help page.

When ERT is activated, the VTP device is divided into 2 logical areas, one for regular simulation execution and one for simulation execution under real-time conditions (ERT). The use of ERT during simulation requires adjustments in the CANoe DE product Simulation Setup. Simulation parts that are not executed in the ERT can be used as usual. Simulation parts that are to be executed in the ERT must be inserted into the simulation in the network nodes associated with the ERT. The data transfer between ERT and the regular simulation execution can be configured automatically or selectively.

ERT can be activated on some VTP devices in addition to Distributed Mode or Standalone Mode.

Activation of Extended Real Time

Two steps are necessary for ERT activation:

Step

Description

1.

Activation on the VTP device

  • Activate via the Vector Platform Manager or CANoe DE product|VTP Tool Platform|Device Settings|Extended Real Time (ERT).
    The VTP device restarts.

2.

Activation in your CANoe DE product

  • Ribbon: File|Options|Extensions|Extended Real Time

  • Activate ERT network node.

Using Extended Real Time

After activation, ERT Network Nodes can be added to the CANoe DE product Simulation Setup.

Note

Only the programs created as ERT network nodes are executed in the ERT. All other network nodes are executed in the regular simulation environment.

Transfer of System Variables

By default, updates of all system variables are synchronized between ERT network nodes and other network nodes. This can have a negative effect on the real-time performance of the ERT network nodes if there are a large number of system variables. A system variable filter can be configured for optimization. This makes it possible to manually define and focus on the required synchronized system variables.

You can configure the system variable filter in the Extended Real Time - Value Transfer dialog.

Supported System Variables

The following system variables are supported in ERT:

  • Integer (32 bit with sign, value range -2.147.483.648 to +2.147.483.647)
  • Integer (32 bit without sign, value range 0 to +4.294.967.296)
  • Integer (64 bit with sign, value range -9.223.372.036.854.775.808 to +9.223.372.036.854.775.807)
  • Integer (64 bit without sign, value range 0 to +18.446.744.073.709.551.616)
  • Double (64 bit IEEE double)
  • Data (variable number of bytes)
  • Integer array (fixed number of 32 bit values with sign)
  • Double array (fixed number of 64 bit IEEE double values)

Note

Structs, strings and generic arrays are not supported by ERT.

Activating CAN modules in Extended Real Time

Some CAN modules can be used either in Extended Real Time or conventionally in the RT kernel. The selection is possible per module via the Vector Platform Manager or CANoe DE product|VTP Tool Platform|Channels/Network Interfaces. By selecting a module, a check box Available in ERT is displayed on the right side. For different devices there are different default settings here. A restart of the device is necessary to change the assignment.
