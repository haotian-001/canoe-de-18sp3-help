Extended Real Time - Value Transfer

Ribbon: Simulation Tab|Extended Real Time|Value Transfer

Valid for:  CANoe DE

In this dialog, you configure the system variable filter.

By default, updates of all system variables are synchronized between ERT network nodes and other network nodes. This can have a negative effect on the real-time performance of the ERT network nodes if there are a large number of system variables. A system variable filter can be configured for optimization. This makes it possible to manually define and focus on the required synchronized system variables.

Note

Only available if you have activated ERT network nodes in the Options dialog.

Global Filter Settings

Updates to system variables are transferred between ERT network nodes and the other network nodes in both directions. The system variable filter can therefore be configured for both transmission directions for both user-defined and system-defined system variables.

CANoe to ERT / ERT to CANoe

The following filter behaviors are available:

  • Pass [On Update]: The system variable value is transferred with every update.
  • Pass [On Change]: The system variable value is only transferred when the value changes.
  • Stop: The system variable value is not transferred.

If you have activated the global filter settings, the variable-specific filter settings are not taken into account.

Individual Variable Filter Settings

In the variable-specific settings, there is a default filter behavior that is initially set for each system variable. You can set this default filter behavior for all existing system variables here.

CANoe to ERT / ERT to CANoe

If you have not activated the global filter settings, the same default filter behavior is available here. The filter behavior selected here applies to all system variables whose filter behavior is set to Default.

List of System Variables

All available system variables are displayed in the table. Here you can adjust the filter behavior for each individual system variable.

To make configuration easier, the filter behavior of all system variables of a namespace can be set via the corresponding namespace. If system variables of a namespace have different filter behaviors, this is displayed in the namespace with the Custom filter behavior.

For a better overview, you can activate the Only show system variables with filter settings differing from Default option to display only those system variables whose filter behavior has been explicitly changed and therefore differs from the default.

You can change the configuration of the system variable filter both before and during a measurement.

Note

From version 18, the configuration of the value transfer replaces the previously possible settings for selective transfer of system variables.

  • If you open a configuration that was saved with a version up to and including 17, the settings for the selective transfer of system variables are adopted in the configuration of the value transfer so that the transfer behavior of the system variables remains unchanged.
  • However, if you save a configuration for a version up to and including 17 in version 18 and higher, the configuration of the value transfer is not translated into the settings for selective transfer of system variables; these are therefore reset to the default settings in versions up to and including 17.
