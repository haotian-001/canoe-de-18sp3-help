Connect Simulink with CANoe

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Basic Concept

The MATLAB Integration Package provides the necessary tools to interface seamlessly between Simulink and your CANoe DE product, while using common mechanism in both domains. In your CANoe DE product, bus signals, system variables, communication objects and distributed objects are used. In Simulink, blocks and signals are available. The MATLAB Integration Package provides a collection of Simulink blocks which allow native access to the respective information in your CANoe DE product.

Transferring Data Between Simulink and your CANoe DE product

You can find the Simulink blocks of the MATLAB Integration Package in the Simulink Library Browser in the area named Vector CANoe. You can find a complete reference of the available blocks on the following pages:

  • I/O Library
  • CO Library
  • DO Library

To use one of the blocks, drag and drop it into the Model. In the following image there is shown a simple composition of two MATLAB Integration Package specific Blocks (System Variable Input, System Variable Output) and one common Simulink Block (Gain).

This model allows to read the value of the CANoe DE product System Variable InputVariable in the CANoeSystemVariable namespace and use the Simulink block Gain to multiply the value with two, before writing it back to the CANoe DE product System Variable OutputVariable in the CANoeSystemVariable namespace.

This mechanism is always the same independent which type of data is accessed. There is an Input Block to read data from your CANoe DE product and there is an Output Block to write the data back. To speed up the modeling process there is a simple way to obtain Symbols, e.g. System Variable names and namespaces, for data elements used in your CANoe DE product. This is described in the next section.

Obtain Symbols From The CANoe DE product

To obtain the Symbols from your CANoe DE product for any type of MATLAB Integration Package Block, e.g. the System Variable Input Block from the example before, double click on the Block. The respective dialog will open and provide details for the current configuration. When clicking on Select variable the Symbol Selection dialog will open and provides a list with all the available Symbols in the currently open CANoe DE product configuration. Clicking OK in the Symbol Selection will apply the Configuration for the selected Symbol.

Note

This mechanism relies on the COM Interface. If no CANoe DE product is open, it will open the COM registered CANoe DE product version. Be aware, if multiple CANoe DE product instances are open, it is not defined from which configuration the symbols are shown.

Using Blocks for Configuration

Besides transferring data, MATLAB Integration Package blocks can also be used for configuration of the model. E.g., the execution mode can partially be configured with the Simulation Step block. All the following blocks allow for configuration:

  • Analysis Step
  • CANoelib Parameters
  • Connect Open Ports
  • Simulation Step
  • XCP Calibration
