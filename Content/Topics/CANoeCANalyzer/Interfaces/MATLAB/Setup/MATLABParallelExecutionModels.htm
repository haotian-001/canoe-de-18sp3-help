Behavior of Models With Concurrent Execution

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

This page describes the behavior of Simulink models built for concurrent execution.

System Requirements

  • One of the compilers listed in the compiler configuration, except Linux, which is not yet supported.
  • DLLs built for concurrent execution require a CANoe Product starting with version 18.

Timing Behavior: Differences between serial and concurrent execution

Models built for serial execution are executed on the main thread. Model execution is considered as atomic step. This means that during execution, the simulation time domain is at a standstill and the output values of the model have the same time stamp as the set execution time of the model.

Models built for concurrent execution are each executed on a separate thread. During model execution, the simulation time continues to run and the output values of the model have a larger time stamp than the set execution time of the model. Reading in all input values and writing all output values is still an atomic step.

Handling of Overruns

If the calculation of a model step is not completed before the next model step has to be calculated, an overrun occurs. The next model step - including reading the inputs - is delayed until the previous model step has been completed. In the CANoelib Parameters block, you can configure the behavior for overruns per model.

  • Stop on overrun (default)

    As soon as an overrun is detected, the overrun counter of the model increases and the simulation is aborted.

  • Enqueue tasks on overrun

    As soon as an overrun is detected, the overrun counter of the model increases and the task is added to the queue. As soon as the previous execution of this model is completed, the task is started in the queue. The maximum number of tasks in the queue can be defined via the Max number of enqueued tasks setting. If more tasks are added to the queue, the simulation is aborted.

You can also change the general behavior of your CANoe DE product for overruns in the Options dialog to configure your CANoe DE product.

Restrictions for Concurrent Model Execution

Currently, only data exchange via signals and system variables is supported.

The following options are not supported and must be deactivated in the code generation options:

  • Export CAPL Functions.
  • Enable Simulink Signal Analysis.
  • Enable Parameterization from CANoe.
  • Parameterization and Signal Analysis also affect referenced models.
  • Exclude specific Blocks from Parameterization.
