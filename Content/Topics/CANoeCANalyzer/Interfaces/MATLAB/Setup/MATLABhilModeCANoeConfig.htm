HIL Mode - CANoe DE product Configuration

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

After code has been generated the model has to be integrated and configured in the CANoe DE product.

In the CANoe DE product, module description files (*.vmodule) can be used to represent different artifacts with their paths and additional metadata. The MATLAB Integration Package uses the module description files to provide an easy way to reference all the previously built artifacts which are required to execut the simulation of a Simulink model in the CANoe DE product . The generation of the module description file can be enabled in CN code generation options. If no output folder is set in the CN code generation options, the generated MODEL.vmodule file can be found in the same folder as the Simulink model.

Note

The module description files created by the MATLAB Integration Package also references a graphical representation of the model. This representation can be viewed in the CANoe DE product, see Simulink Model Viewer for further information.

You can add the Simulink model compiled for HIL-Mode as a Simulation Model into the CANoe DE product, via the Communication Setup or the Simulation Setup (only valid for CANoe DE, CANoe:lite DE).

To add an analysis model in the Simulation Setup follow the steps described in Simulink Analysis Models.

Simulation Setup (only valid for CANoe DE, CANoe:lite DE)

To add a Simulink model compiled for HIL mode to the Simulation Setup, a network and a network node are required. Use a normal Network Node to execute for Standard Realtime. For Extended Real Time choose an ERT Network Node.

Configure the Simulink model in the Network Node, by adding the module description file to the Components tab.

Communication Setup

To add a Simulink model compiled for HIL mode into the Communication Setup, add the module description file as an application model.

Note (only valid for CANoe DE, CANoe:lite DE)

At the moment this requires an existing network in the Simulation Setup.

Legacy Configuration

Note

The legacy Simulink simulation dialog is only remaining for compatibility reasons and will be removed in a future version of the CANoe DE product. Rely on the methods described above to configure Simulink Models in the CANoe DE product.

First of all the Simulink Legacy support of the CANoe DE product must be configured. Open the Options dialog (External Programs|Simulink) and activate the Legacy Simulink Integration check box. The Simulink model itself is configured in the context of a node in the Simulation Setup. Open the Simulink page of the configuration dialog of the node to which you want to add the model. Specify the model file, the INI file and the DLL as requested, see image below.

Now the model will be calculated in the simulation context of the CANoe DE product and the model can be viewed with the Model Viewer:
Environment ribbon tab|Simulink Viewer|<name of the model>.

Notes

  • The model file and INI file are necessary to display the Simulink model with the Model Viewer. If you don’t have them because you switched of the appropriate code generation setting, the model will be calculated nevertheless because you have configured the DLL.
  • For Models compiled with ERT support, please use an ERT node. These ERT nodes must be first activated in the Options dialog under Measurement|Performance.
