ASAM XIL API

The ASAM XIL API is a standard that specifies software interfaces for access to test systems. This API allows different test systems to be programmed in a standardized manner. Furthermore, investments in test scripts remain protected even when a change is made from one test system to another.

This component is included in the setup of your CANoe DE product. In the setup folder Installer Additional Components you find the corresponding setup files.

CarMaker Interface

Via the CarMaker interface it is possible to start single tests or test series in CarMaker and monitor selected simulation data.The control and stimulation of the CarMaker test runs is done via the APO interface provided by CarMaker. This can be called from your CANoe DE product using special CAPL functions.

This component is included in the setup of your CANoe DE product. In the setup folder Installer Additional Components you find the corresponding setup files.

FDX Protocol

FDX (Fast Data eXchange) is a protocol that can be used for easy, fast, and real-time data exchange between your CANoe DE product and other systems via an Ethernet connection. The protocol gives other systems read/write access to system variables, environment variables, and bus signals of the CANoe DE product. In addition, it is possible to send control commands (e.g., for starting and stopping the measurement) to the CANoe DE product or to receive status information via FDX.

Functional Mock-up Interface (FMI)

The Functional Mock-up Interface (FMI) defines a standardized interface for the coupling together of complex dynamic simulation models from various modeling tools (e.g., Simulink). One of its purposes is to enable and simplify validation and testing of the interaction of components within a simulated environment.

LabVIEW Integration

By using the ActiveX library that is included in LabVIEW, your CANoe DE product can be activated and addressed via the COM Interface. Therefore the COM Interface ensures that all objects (and their methods, properties and events) are available in LabVIEW.

This component is included in the setup of your CANoe DE product. In the setup folder Installer Additional Components you find the corresponding setup files.

MATLAB Integration

The purpose of this interface is to extend the node modeling capability of your CANoe DE product by adding the strength of the MATLAB/Simulink environment. Now models can be used to describe the functionality of a network node in the CANoe DE product. This allows the CANoe DE product to simulate nodes which are impossible or difficult to model using CAPL alone.

This component is included in the setup of your CANoe DE product. In the setup folder Installer Additional Components you find the corresponding setup files.
