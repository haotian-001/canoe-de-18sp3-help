Multi-Application Capability of the Network Interfaces

 

Valid for:  CANoe DE  •  CANoe:lite DE

General

The CAN and FlexRay channels of the Vector network interfaces can be used by several applications simultaneously (e.g. by CANoe and CANape).

Notes

  • You can start the applications in any order.
    Please note, however, that the first application needs to configure the channels.
  • The first application that uses the interface channel obtains Init Access.
    You can see this in the Vector Hardware Config applet, where the application channel status is set to Init.
  • Full functionality is available only to the application that has Init Access.
    The limitations imposed on applications that do not have Init Access depend on the bus system used.

CAN

  • The channel can only be configured (baudrate, BTR) by the application that has Init Access. CAPL functions that modify the channel configuration are not executed.

FlexRay

  • If a channel does not have Init Access when measurement starts, a warning message appears in the Write Window. As with CAN, the channel is configured only by the application that has Init Access. Channel configuration includes:
  • Loading of cluster parameters
  • Configuration of the Symbol Window
  • Configuration of the key slots (sync frames)
  • Automatic restart if synchronization fails
  • Configuration of the wake-up pattern
  • Setting the channel mode (synchronous, asynchronous mixed)
  • You do not need to modify CANoe and CANalyzer configurations that you intend to run without Init Access. The exception here is the channel mode. If the mode does not correspond to the current channel mode, the measurement is canceled. An error message appears in the Write Window.
  • Frame and PDU configuration
  • Init Access is not needed to send frames and PDUs. However you can configure one slot to serve exclusively as the send slot for one (1) application.
  • CAPL functions that modify the channel configuration can only be run with Init Access. This includes e.g. frSetMode, frSetConfiguration, frSetExtOffsetRateControl, frSetKeySlot, frSetPOCState, ResetFlexRayCC etc. Without Init Access, these functions are ignored and have no effect.

CAPL Programs on the Interface Hardware

  • CAPL programs can only run with Init Access.
    Without Init Access, the measurement start is canceled and an error message appears in the Write Window.
