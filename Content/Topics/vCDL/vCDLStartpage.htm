Vector Communication Description Language (vCDL)

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

vCDL is a tool for the simple, text-based configuration of application layer objects.

If you are new to vCDL, it is recommended to read the help page by page to get an overview of its capabilities. The syntax is similar to well-known languages like C++, C#, and Java — with some deviations. Its purpose is data modeling only thus missing procedural elements such as loops. Nevertheless, attributes can impact the behavior of the modeled data during measurement.

If you have already worked with vCDL in the past, you have most likely done so with vCDL 1.4 or older. vCDL 2.0 is compatible with vCDL 1.4 in most respects, but brings some changes. vCDL 2.0 supports only distributed objects, for example. It is best to start with an existing vCDL file and change the version in the first line to 2.0. You can then read the meaning of the errors output by the version adjustment in the help.

Why the introduction of version 2.0?

When vCDL 1.0 was released back in 2018, several design choices were based on assumptions as no language with similar purpose had existed before. Over the next years many new features were added while vCDL was kept backwards compatible posing restrictions on new designs. Additionally, the vCDL compiler had difficulties keeping up with files continuously growing in size.

All these aspects combined led to the decision to create a revised major version of vCDL. Our main goal was removing as many design flaws and inconsistencies as possible while keeping it close to the original vCDL. This of course led to breaking changes meaning that some vCDL 1.x files require changes to be used as vCDL 2.0. Rest assured that each breaking change has been analyzed extensively before considered necessary to improve the overall design of vCDL 2.0. On the other hand, the redesign of the vCDL 2.0 compiler enables various new features that would not have been possible with a potential vCDL 1.5 version.

It is recommended to create new vCDL files with version 2.0.

Example

version 2.0;

import module "Abstract";

namespace System
{
  struct Wheel
  {
    float Pressure;
    int   RPM;
  }

  [Binding="Abstract"]
  interface ICar
  {
    provided data Wheel FrontLeft;
    provided data Wheel FrontRight;
    provided data Wheel RearLeft;
    provided data Wheel RearRight;

    provided field int SteeringAngle = 0;

    consumed data bytes TrafficData;
  }

  typedef ICarStimulator = reverse<ICar>;

  ICar RedCar;

  ICarStimulator CarStimulator;
}
