Sample Configurations I/O HIL

 

Valid for:  CANoe DE  •  CANoe:lite DE

Note

Click on the blue link to load the sample configuration in your CANoe DE product.

Note

If there is no description for the sample configuration in the help, you find a description in a TXT file next to the sample configuration in the same folder.

After you have loaded the sample configuration in your CANoe DE product you can show this TXT file via the Environment ribbon tab|Documents.

General

Easy (Ethernet - FDX)

This sample consists of two parts:

  • Easy (Ethernet - FDX) – EasyFDX.cfg
    Demonstrates the usage of the FDX protocol.
  • external program EasyClient.exe – Start Program

View Description

Map and Video Window – GPS.cfg
Shows the usage of the GNSS function and the Video function.

View Description

TCP/UDP Client and Server

  • TCP/UDP Client – Tcp_Udp_Client.cfg
    Shows the usage of the TCP/IP CAPL API Client in your CANoe DE product.
  • TCP/UDP Server – Tcp_Udp_Server.cfg
    Shows the usage of the TCP/IP CAPL API Server in your CANoe DE product.

View Description

RS232 – RS232.cfg
Demonstrates the access to the RS232 serial port.

View Description

IOpiggy – IOPiggy.cfg
Demonstrates the use of IOpiggy 8642 and the control via System Variables.

—

I/O Hardware – IO_Hardware.cfg
The I/O network interface supports reading and writing of analog and digital I/O ports of various I/O cards using system variables.

View Description

General Purpose Interface Bus – GPIB.cfg
Shows the necessary CAPL code required to connect your CANoe DE product to an external device via GPIB.

View Description

Extended Real Time

—

System Configuration (Extended Real Time) – ERTSystem.cfg
Demonstrates the execution of network nodes in a real-time environment using the ERT feature.

View Description

Extended Real Time - Compare To RT – ERTComparisonWithRT.cfg
Direct comparison between a MATLAB model execution in distributed mode and with Extended Real Time.

View Description

VT System

The following sample configuration demonstrate the VT System.

—

VT System - Online – VTDemoOnline.cfg
Set up and check VT modules.

VT System - Offline – VTDemoOffline.cfg
Use different VT System modules to test ECUs.

View Description

VT2848 Crank and Cam Shaft Stimulation – VT2848CrankCam.cfg

VT System User FPGA

The following sample configurations show the usage of the User FPGA for dedicated VT System modules.

—

VT2848 Crank and Cam Shaft Stimulation (FPGA) – VT2848CrankCamFPGA.cfg
Stimulation of a crank and a cam shaft signal with the User FPGA of the VT2848.

View Description

VT System Basic User FPGA – VTSBasicUserFPGA.cfg
Shows how the User FPGA for works for different VT System modules.

View Description

VT7820 User FPGA – VT7820Demo.cfg
Shows the output of four wheel speed sensors with 50% duty cycle based on a moving car.

View Description

VT2710 SPI User FPGA – VT2710SPIUserFPGA.cfg
Shows how an SPI sensor can be implemented in the User FPGA of the VT system.

View Description

VT2710 I2C User FPGA – VT2710I2CUserFPGA.cfg
Shows how to implement an I2C sensor in the User FPGA of the VT System.

View Description

VT System Inter Board Communication – VTSInterBoardCommunication.cfg
Shows the direct connection of several VT System User FPGAs via ring bus.

View Description

LabVIEW

Demonstrates how to use LabVIEW with CANoe using network variables. A LabVIEW installation is required.

This sample consists of two parts:

  • LabVIEW project – Open Folder
  • LabVIEW – LabVIEW.cfg
    Demonstrates how to couple your CANoe DE product with LabVIEW.

View Description

Bouncing Ball (FMI) – BouncingBall.cfg
Demonstrates the Functional Mock-up Interface (FMI) with a connected Functional Mock-up Unit (FMU).

View Description

Easy (FMI)

This sample consists of several parts:

  • Easy (FMI) – EasyFMI.cfg
    Provides the CANoe DE product Configuration as FMU Functional Mock-up Unit via FMI Functional Mock-up Interface.
  • FMU Consumer – FMUConsumer.cfg
    Uses the FMU provided from the sample configuration Easy (FMI) and serves their inputs and outputs.
