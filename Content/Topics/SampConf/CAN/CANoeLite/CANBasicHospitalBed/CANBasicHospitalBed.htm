Basic Hospital Bed (CAN)

 

Valid for:  CANoe:lite DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • Basic Hospital Bed – CANBasicHospitalBed.cfg
    Shows basic CAN functionality and a test unit that controls and tests a hospital bed.

Introduction

The sample configuration contains a simple simulation of a hospital bed.

After starting the measurement, you can control the hospital bed in the Control Panel using the Function Pendant. Changes to the height or head position of the bed are displayed in the Display Panel (desktop Hospital Bed). A test unit can automatically control and test the bed (desktop Test).

Desktops

This configuration contains the following desktops:

Hospital Bed

Analysis

Test

Configuration

 

Hospital Bed [▲ back]

This desktop shows an overview of the hospital bed simulation. The simulation is visualized and controlled via the following two panels:

  • Display Panel

    Displays the current status of the bed. This panel is for visualization only.

  • Control Panel

    Here you can control the bed height, head position and bed lighting using buttons on the function pendant.

In addition to the Display Panel, you can also track changes to the bed status on this desktop in the Data and Trace windows.

Analysis [▲ back]

On this desktop, you can control the simulation with the Control Panel (see description of Hospital Bed desktop).

You can observe changes to the simulation state in the Data, Graphics and CAN Statistics windows.

Test [▲ back]

The Bed Test Configuration test unit is located on this desktop. When you start the test, it takes control of the bed and checks whether its state changes as expected.

You can observe the automatic control in the Display Panel (see description of the Hospital Bed desktop).

By default, all test cases will pass. You can use the Error Panel to provoke errors that will cause various test cases to fail.

In the Test Trace 1 window, you can observe which test steps are currently being executed.

Configuration [▲ back]

This desktop shows the Simulation Setup. The simulation of the bed is implemented via a CAPL script on the HospitalBed simulation node. The ControlUnit simulation node represents the ECU connected to the bed via the CAN channel.
