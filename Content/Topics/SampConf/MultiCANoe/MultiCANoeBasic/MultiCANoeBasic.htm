Basic Configuration (Multi CANoe)

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • Basic Configuration (Multi CANoe)

    • CANoe DE: CoordinatorConfig.cfg
    • CANoe DE: AgentConfig.cfg

    Demonstrates how to use two of your CANoe DE product instances in a Multi CANoe cluster.

Introduction

Load the two configurations CoordinatorConfig.cfg and AgentConfig.cfg in their respective instances of your CANoe DE product on the same computer. The Multi CANoe cluster is configured to use the localhost connection. Thus, if both configurations are loaded, the Multi CANoe cluster state should switch to Connected. The measurement can be started from any instance, and the Multi CANoe feature ensures that the measurement is also started in the other instance.

Multi CANoe can only be executed in Real Bus mode. The sample configurations require one CAN channel each. If no CAN hardware is available, the CAN1 channels can also be assigned to a virtual CAN bus.

The Multi CANoe feature can also be used to couple CANoe DE product instances running on different devices. For this, you must change the Multi CANoe cluster settings in the global options.

System Variables

The Multi CANoe instances use system variables to communicate between the two instances. You can find more information about the configuration of system variables for Multi CANoe on page Create a Multi CANoe Environment under Data Exchange Between CANoe DE Instances.

Overview [▲ back]

This desktop contains the MultiCANoePanel, which demonstrates some of the typical functionality of Multi CANoe:

  • Start System

    This switch toggles the processing of the counters on and off.

  • Execute Tests

    This switch performs distributed tests in both Multi CANoe instances. The CoordinatorTest and the AgentTest start synchronously and execute their tests sequentially, waiting for the previous test step to finish.

  • Counter in Coordinator / Agent and Shared Counter in Coordinator and Agent

    This options allow you to manually activate counters in the specified instances.

  • Counter Settings

    This area allows you to set a Count Delay in milliseconds, which defines the interval between two counter increments.

  • Send Simultaneous Events

    This button triggers a timer event with the same simulation time in both Multi CANoe instances, sending a CAN frame and writing a system variable at the same time.

The test module windows CoordinatorTest and AgentTest allow you to follow the test execution.

The Trace window shows all sent and received CAN frames and all system variable updates.

The Write window displays information, warnings, and errors that occur during measurement.

Setup [▲ back]

This desktop contains the windows Simulation Setup, Measurement Setup, and Test Setup.

The Simulation Setup in the CoordinatorConfig.cfg and AgentConfig.cfg contain the following imulation nodes:

  • CoordinatorCounter / AgentCounter

    These nodes contain a CAPL program that implements a simple counter for a system variable with Multi CANoe write access set to this instance, allowing only one instance to write the system variable. The other instance can access the system variable as read-only. The counting can be enabled from both instances.

  • CoordinatorAllCounter / AgentAllCounter

    These nodes contain a CAPL program that implements a shared counter for a system variable with Multi CANoe write access set to All instances. The coordinator increases the counter for even values, while the agent increases the counter for odd values.

  • CoordinatorSimultaneousEvent / AgentSimultaneousEvent

    These nodes demonstrate how to create a timer in several Multi CANoe instances that fires at the same time.

The Simulation Setup in the CoordinatorConfig.cfg contains additionally the simulation node Init:

  • Init

    This node shows how to send a CAN frame once the Multi CANoe cluster is connected.

The Measurement Setup is configured very minimally, containing only a Trace window.

The Test Setups contain one test module each (the AgentTest and the CoordinatorTest). These test modules start their tests simultaneously by using a start trigger on a system variable. The tests use a common system variable with Multi CANoe write access set to All instances, which they use to determine which test instance should execute the next test.
