System Configuration (ISO11783)

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

    Simulation of a ISO11783 total system.

Introduction

The sample configuration simulates a complete ISO11783 network. This includes the following components:

  • a tractor
  • a Task Controller
  • a Virtual Terminal
  • a sprayer
  • a GPS receiver

At the start of the measurement, all ISO11783 components log on to the bus. The sprayer transmits its object pool to the Virtual Terminal (VT) and its device description to the Task Controller (TC). An internal test of the sprayer is started via the soft key Test on the VT. The task is then activated on the Task Controller and the sprayer is set to working mode via the on soft key on the VT.

The tractor is set in motion using the accelerator pedal in the Tractor panel. The Task Controller controls the sprayer by setting the application rate via the Setpoint process variables and switching individual nozzles on and off. There are three treatment zone, each of which is represented on the map by its own yellow polygon. Before entering the treatment zone, the predefined application rate for the respective treatment zone is set and individual nozzles are switched on when the boundary is passed. The sprayer confirms the set values and the current status of each individual nozzle via the corresponding Actual process variables. The route traveled and the set and confirmed operating mode of each individual nozzle are also displayed on the map.

Auto Play

The first time the configuration is loaded, Auto Play mode is activated. In this mode, all user input is created automatically by the simulation:

  • Test the sprayer
  • Activate the task
  • Setting the sprayer to work mode
  • Controlling the tractor

In the Control panel, the actions that have not yet been performed are displayed by red LEDs and the current actions by green LEDs.

Manual Control

In the Control panel, you can switch off the Auto Play mode. In this case, you must make all user entries yourself. The next pending action is displayed in the same panel by a green LED. When the action is completed, the color of the LED changes to white. After the initialization phase (executing the test in the Virtual Terminal and starting the task in the Task Controller), you can change the speed of the tractor (using the pedals in the Tractor panel) and switch the sprayer on and off (using the on and off soft keys in the VT).

Setup [▲ back]

This desktop shows the Simulation Setup and the Measurement Setup.

The Measurement Setup contains blocks that can be used to analyze the data traffic on the CAN bus.

The Simulation Setup contains the simulated ECUs:

The Task Controller is simulated by the simulation node pair TC and TC Window, the Virtual Terminal by the simulation node pair VT and VT Window. The simulation node Tractor is responsible for the tractor, the simulation node Sprayer for the sprayer. The simulation node GpsReceiver creates GNSS parameter groups that simulate the route. The simulation node MapHelper shows three polygons that represent the treatment zones.

Simulation [▲ back]

On this desktop the following windows are displayed:

Window

Description

VT Window

The window shows the object pool of the sprayer via which it is controlled.

TC Window

The window shows the device description of the sprayer with all process variables, their current values and permitted triggers. The Process data table view shows the progression over time of the process variables Setpoint App Rate, Actual App Rate, Setpoint Cond Work State and Actual Cond. Work State.

Two scenarios are preconfigured in the Scenarios view :

  • Enter TreatmentZone

    Simulates entering a treatment zone. The Setpoint App Rate is set and the individual nozzles are activated one after the other.

  • Leave TreatmentZone

    Simulates leaving a treatment zone. The individual nozzles are switched off one after the other and Setpoint App Rate is reset.

Map Window

The window shows the treatment zones, the tractor route and the set (red) and confirmed (green) operating statuses of all nozzles.

Tractor

The panel displays the current speed and direction of travel. The speed can be adjusted using the Gas and Brake pedals. The direction of travel cannot be influenced.

Control

You can switch the Auto Play mode on and off via the panel and call up the description of the configuration in the help. The LEDs display which actions have been executed (white), which can currently be executed (green) and which are not yet allowed (red).

Trace Filtered

The most important parameter groups are displayed here, which represent the data exchange between the sprayer, Virtual Terminal and Task Controller. Filtering is carried out via the upstream CAPL node Trace Filter and the two activated analysis filters on the left side.

Trace [▲ back]

The Trace All and Trace Filtered windows are displayed on the desktop:

  • Trace All shows all parameter groups that appear on the CAN bus. You can use the analysis filters on the left side to limit the number of parameter groups displayed.
  • Trace Filtered only shows the parameter groups that represent the data exchange between the sprayer, Virtual Terminal and Task Controller. Of the status messages Working Set Maintenance, VT Status, TC Status and TC Client Task, only those that signal a new status are displayed. Filtering is carried out via the upstream CAPL node Trace Filter and two activated analysis filters.
