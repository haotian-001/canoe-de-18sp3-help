Basic AUTOSAR SOME/IP Event

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • Basic AUTOSAR SOME/IP Event – SOMEIPBasicAutosar.cfg
    Shows how the AUTOSAR Ethernet interaction layer (CANoeILNL_AUTOSAR_Eth.vmodule) is used. The sample configuration shows how to use SOME/IP events, fields and methods.

An ARXML file is assigned to the CANoe DE product configuration. It describes the two ECUs CAMF and ADAS. The CAMF ECU offers a CameraObjectDetection service. This is subscribed to by the ADAS ECU. The service comprises the two events LaneDetection and TrafficSignDetection, the field FrameRate and a method StartStop. Open the Symbol Explorer and select the Application Layer Objects group to display the service elements.

The Simulation Setup contains the two simulation nodes CAMF and ADAS. The simulated nodes use two interaction layer VMODULE files:

  • AsrPDUIL.vmodule
    This is responsible for the sending behavior, e.g. cyclic sending.
  • CANoeILNL_AUTOSAR_Eth.vmodule
    This IL transmits the PDUs and performs the service discovery.

Both interaction layers are configured from the ARXML.

Introduction

At the start of measurement, the CameraObjectDetection service is offered by the CAMF simulation node and subscribed to by ADAS. The control of the simulation is based on SOME/IP application objects. These objects are automatically generated by your CANoe DE product when the configuration is loaded (see Setting CANoe options | Bus systems /Protocols | Ethernet).

Desktops

This configuration contains the following desktops:

Setup

Measurement

Setup [▲ back]

The Simulation Setup contains the two simulated nodes CAMF and ADAS. Both use a simple CAPL program to enable the CameraObjectDetection service to be offered/subscribed via the panels. The CAPL programs also contain sample code that illustrates the handling of application objects (focus on event handlers).

The main work is handled by the interaction layers, which are assigned via the Node Configuration dialog on the Components page. In the Start Values window, the system variables for activating the service are set in such a way that the service is subscribed to immediately at the start of measurement.

Measurement [▲ back]

The two panels for controlling the simulation nodes CAMF and ADAS are located on this desktop. The messages exchanged between the two simulation nodes are displayed in the Trace window after the measurement has started. Both the application objects and the SOME/IP messages including service signals are displayed.

The two simulation nodes can be stimulated via the two panels CAMF - Provided Service and ADAS - Consumed Service. Application objects are assigned to the panel controls. You can trigger events and call methods via the panels.

Example

  1. Start the measurement in your CANoe DE product via Home | Start.
  2. The system automatically switches to the desktop Application. Select the desktop Measurement at the bottom of your CANoe DE product to switch back to this desktop.
  3. In the Trace window, you will see the Service Discovery messages. These are of no interest for the further process, so delete the contents of the Trace window using the Clear Trace window contents button in the toolbar of the Trace window.
  4. Select any value in the DetectedTrafficSign[0] selection field of the CAMF - Provided Service panel.
  5. The following display appears in the Trace window:

Note

For reasons of clarity, the rows in the Trace window have not been expanded. Click on the Trace button in the toolbar of the Pause window and then expand the row entries to obtain further information.

Line 1: Shows the value of the application object of the service provider CAMF.

Line 2: Shows the Ethernet packet in which the SOME/IP notification is sent.

Note

In simulated mode, all simulation nodes are connected to one switch. The switch offers a port with a corresponding name for each simulation node. The Ethernet packet was sent from the simulation node CAMF and received at the port CAMF (direction = Rx).

Line 3: Shows the SOME/IP notification extracted from the Ethernet packet (line 2).

Line 4: Displays the Ethernet packet that was sent via the port ADAS to the simulation node ADAS (Direction = Tx). This is the same Ethernet packet that has already been measured on port CAMF.

Line 5: Shows the SOME/IP notification extracted from the Ethernet packet (line 4).

Line 6: Shows the value of the application object of the service consumer ADAS.

Note

Using suitable filters (column filters, predefined filters or analysis filters), you can customize the display in the Trace window to suit your requirements.

The Enable Service CameraObjectDetectionPSI1 check box in the CAMF - Provided Service panel can be used to switch off the service in the CAMF simulation node. The check box Subscribe Servcie CameraObjectDetectionSCI1 deactivates the service registration in the simulation node ADAS.

Application [▲ back]

At the start of the measurement, the system switches to this desktop. The transmitted service signals are displayed here in the Trace window and in the Data window.

The two application panels are located on this desktop:

  • Vehicle_Ethernet::CAMF::CameraObjectDetectionPSI1
  • Vehicle_Ethernet::ADAS::CameraObjectDetectionCSI1

These are generic panels. These are automatically created by your CANoe DE product. You can open the application panels via Simulation | Application Panel. All services offered and subscribed to by the simulation nodes, their methods, fields and events are displayed in the panels.

Example

  1. In the application panel Vehicle_Ethernet::ADAS::CameraObjectDetectionCSI1, select the method StartStop.
  2. In the right-hand area of the Application Panel, set the value of the method parameter Activate.
  3. Now activate the Call button.

You can then see the method call and response in the Trace window. The return parameters are displayed on the Response tab in the Vehicle_Ethernet::ADAS::CameraObjectDetectionCSI1 application panel.

Note

Application objects are created for both the service provider and the service consumer. The method parameters and return values are therefore available in both application panels.

The Data window displays both application objects and service signals.

Example

  1. Restart the measurement.
  2. Select the desktop Measurement.
  3. In the CAMF - Provided Service panel, deactivate the Enable Service CameraObjectDetectionPS check box.
  4. Select the desktop Application.
  5. Change any value in the event TrafficSignDetection of the application panel Vehicle_Ethernet::CAMF::CameraObjectDetectionPSI1.
  6. Activate the Trigger button for the TrafficSignDetection event.

In the Data window, the value you have set is displayed for the associated application object. As the service is not currently offered (the check box Enable Service CameraObjectDetectionPSI1 has been deactivated), no event notification is sent. The service signal associated with the application object has therefore not been updated. The old value is still displayed in the Data window.
