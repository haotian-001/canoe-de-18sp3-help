Basic AUTOSAR I-PDU

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • Basic AUTOSAR I-PDU – PDUBasicAutosar.cfg
    Shows how the AUTOSAR Ethernet interaction layer (CANoeILNL_AUTOSAR_Eth.vmodule) is used to send and receive an I-PDU.

In the ARXML database, a PDU is configured that is sent cyclically from the VGW simulation node to the IC simulation node. The simulated nodes use two interaction layer VMODULE files:

  • AsrPDUIL.vmodule
    This is responsible for the sending behavior, e.g. cyclic sending.
  • CANoeILNL_AUTOSAR_Eth.vmodule
    This IL transmits the PDUs and performs the service discovery.

Both interaction layers are configured from the ARXML database.

Introduction

When the measurement starts, the EnginePSI1 service is offered by the VGW simulation node and consumed by the IC simulation node. This is followed by the cyclical transmission of the PDU EMotorStatus.

The simulation nodes are stimulated using application objects. Their values are changed via Signal Generators. The panels are used to visualize the service signal values.

Setup [▲ back]

The Simulation Setup contains the two simulation nodes VGW and IC. Both use a simple CAPL program to control the activation of the service via the panels. The main work is handled by the interaction layers, which are assigned via the Node Configuration dialog on the Components page.

In the Start Values window, the system variables for activating the service are set in such a way that the service is subscribed to immediately at the start of measurement.

The Measurement Setup contains the CAPL program PDU Observer that monitors the transmitted signal values. In this CAPL program, an on pdu EMotorStatus handler is implemented, which outputs a message on the PDUObserver panel (Desktop Simulation) if a limit value is exceeded.

Simulation [▲ back]

At the start of measurement, a changeover to this desktop occurs. The application objects or the transmitted signals are displayed here in the Graphics window and Data window. If the Signal Generators are activate, a sine curve to the corresponding signal is output in the Graphics window.

Using the Enable Service (sif_1001) check box in the VGW panel, you can disable the service in the VGW simulation node. The check box Subscribe Servcie (sif_1001) deactivates the service registration in the simulation node ADAS.

Example

  1. Restart the measurement.
  2. Deactivate the Enable Service (sif_1001) check box in the VGW panel.

The EnginePSI1 service is no longer offered. The cyclical transmission of the PDU EMotorStatus is therefore stopped. In the Data window, you can see that the value of the application objects continues to change. As the cyclical transmission of EMotorStatus stops, the service signals, which are also contained in the Data window, are no longer updated.

The same effect is shown in the Graphics window, which contains both an application object and a service signal.

Trace [▲ back]

The Trace window shows service discovery frames and the cyclic transmitted EMotorStatus PDU.

Row 1: Shows the values of the application objects of the service provider VGW. The values are changed cyclically by the Signal Generators.

Row 2: Shows the Ethernet packet in which the AUTOSAR PDU is sent.

Note

In simulated mode, all simulation nodes are connected to one switch. The switch offers a port with a corresponding name for each simulation node. The Ethernet packet was sent by the simulation node VGW and therefore received at port VGW (Direction = Rx).

Row 3: Shows the AUTOSAR PDU EMotorStatus extracted from the Ethernet packet (line 2).

Row 4: Displays the Ethernet packet that was sent via the port IC to the simulation node IC (Direction = Tx). This is the same Ethernet packet that has already been measured at port VGW.

Row 5: Shows the AUTOSAR PDU EMotorStatus extracted from the Ethernet packet (line 4).

Row 6: Shows the values of the application objects of the service consumer IC.

Note

Using suitable filters (column filters, predefined filters or analysis filters), you can customize the display in the Trace window to suit your requirements.
