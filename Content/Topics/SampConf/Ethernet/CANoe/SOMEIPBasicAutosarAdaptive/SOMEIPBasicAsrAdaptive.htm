SOME/IP Basic AUTOSAR Adaptive

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • SOME/IP Basic AUTOSAR Adaptive – SOMEIPBasicAsrAdaptive.cfg
    Uses SOME/IP and application objects (distributed objects).

The sample configuration is based on an AUTOSAR Adaptive database. It describes a Calculator service interface with the communication participants Consumer and Provider.

The Calculator service interface contains four methods, that correspond to the four basic arithmetic operations. In addition, it contains an event that transfers the number of calls made for each of the four methods.

The Consumer communication participant uses the Calculator service interface. The Consumer calls the methods and receives the event.

The Provider communication participant provides the Calculator service interface. The Provider implements the four methods and sends the event.

All communication participants are simulated in your CANoe DE product. The simulation uses SOME/IP and application objects (distributed objects).

Desktops

Trace

Configuration

Measurement

 

Trace [▲ back]

Via the Consumer panel you can stimulate the Consumer communication participant.

Via the separated controls you can call the four methods of the Calculator service interface. Via the Request page you can set the In-Paramenters. The method call you can start with the Send button. The result is shown on the Response page.

Via the Trace Window you can watch the communication on application layer. The method calls and transmitted events between Consumer and Provider are displayed there.

Configuration [▲ back]

The individual elements of the State event are visualized in a Graphics window and a Data window. E.g. you can drag and drop the application layer objects CalculatorConsumer.Add.operand1, CalculatorConsumer.Add.operand2, CalculatorConsumer.Add.result and CalculatorConsumer.State.AddCount from the Symbol Explorer to the Data window and look at its values.

Measurement [▲ back]

Simulation Setup contains AUTOSAR Adaptive database in Databases.

The implemenation of the behavior of events and methods is shown as CAPL code. This is based on the distributed application objects.
