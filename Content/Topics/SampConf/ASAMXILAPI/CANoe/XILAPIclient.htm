ASAM XIL API - Client

 

Valid for:  CANoe DE  •  CANoe:lite DE

Click on the blue link to load the sample configuration in your CANoe DE product.

This sample consists of several parts:

  • ASAM XIL API – Client

    • CANoe DE: XILAPIClient.cfg
    • CANoe:lite DE: XILAPIClient.cfg
    • CANoe4SW DE: XILAPIClient.cfg
    • CANoe4SW:lite DE: XILAPIClient.cfg

    Demonstrates how to access third party XIL API assemblies from CANoe DE product test units.

  • XIL API Server – Open Folder
    Contains a .NET assembly, which simulates a simple XIL API server.
  • XIL API Configuration – Open Folder
    Contains the test bench and port configuration files for the example assembly.

This sample configuration demonstrates the use of the XIL binding to connect systems from other manufacturers via the ASAM XIL interface and to read and write variables

Setup

The XILAPIServer folder contains an exemplary testbench implementation of the ASAM XIL interface. The implementation defines two variables (one readable and one writable variable). The readable variable always has the value of the writable variable plus "1". Exactly this behavior is to be tested with a simple automated test. The source code is included if you want to make adjustments and recompile the testbench.

The test description can be found in the Test folder. To show that the ASAM XIL interface can be used with this solution not only via C#, the same test is implemented in the languages C#, CAPL and Python. An installed Python interpreter in the 64-bit version is required for the Python tests, which is why these test cases are commented out.

Workflow for Generating the vCDL File (optional)

This step is not necessary. The example can be executed directly.

If you want to reproduce the workflow for generating the vCDL file with the xil-vcdl-generator command line tool, you can use the config.xil-vcdl.yaml. To do this, overwrite the existing vCDL file, which is located in the DataSources directory. The working directory must correspond to the config.xil-vcdl.yaml directory so that the relative paths for loading the testbench implementation are correct.

To run the sample configuration, you must switch to the Automated Tests desktop and start the test unit execution.

Setup [▲ back]

The Communication Setup contains all configuration settings. The vCDL file is referenced here.

Analysis [▲ back]

The Desktop Analysis contains the Trace Window. This visualizes the value transfers between your CANoe DE product and the XIL testbench.

Automated Tests[▲ back]

The Automated Tests desktop consists of a test configuration and the Test Trace Window.

Start the execution of the automated tests by clicking the Start button in the test configuration.

During the test execution, you can observe all actions in the Test Trace Window. Once the tests have been completed, all test results and a detailed report on the test are available. Open this via the Open test report button in the test configuration.
