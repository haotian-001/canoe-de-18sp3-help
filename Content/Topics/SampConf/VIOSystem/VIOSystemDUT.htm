VIO System DUT

 

Valid for:  CANoe DE  •  CANoe:lite DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • CANoe DE: VIOSystemDUT.cfg
  • CANoe:lite DE: VIOSystemDUT.cfg

Operation of a VIO System with real hardware as a Device Under Test (DUT).

The sample configuration VIO System DUT illustrates the operation of a VIO System with real hardware as a Device Under Test (DUT).

The hardware consists of the following components:

  • An Arduino MEGA2560 microcontroller board.

  • A TFT display (2,2 inch, 220x176 pixels) to display some system information and the operating status.

  • A 4-bit counter whose speed can be changed via an adjustable voltage.

  • A dimmable LED that is controlled via an analog voltage. The voltage is set via a virtual slider in your CANoe DE product (Analog Dimmable LED).

  • A small fan that is controlled via a PWM signal and whose speed is measured via a PWM signal. The control is set via a virtual slider in your CANoe DE product (Fan Speed Control).

  • A PWM-dimmable LED that is controlled via the duty cycle of a PWM signal. The duty cycle is set via a rotary control on the Device Under Test (PWM Dimmable LED).

  • An ultrasonic distance sensor.

    The signal propagation time is converted into a distance in the Arduino and displayed in your CANoe DE product.

  • A digital temperature sensor that is controlled and read out via I2C.

    (As long as the I2C functionality is not yet available in VIO System, this sensor is inactive)

  • An analog temperature sensor whose temperature is recorded via the VIO1008 voltage measurement card.

    (As long as the I2C functionality is not yet available in VIO System, this sensor replaces the I2C temperature sensor)

  • A real-time clock module that is read out via SPI.

    (As long as the SPI functionality is not yet available in VIO System, the system time of the computer is displayed)

The VIO System consists of the following I/O cards:

  • VIO2004: Stimulation card with 4 channels. The VIO2004 card:

    • Generates the control voltage for the speed of the counter.

    • Generates the voltage for the analog dimmable LED.

  • VIO4028:Digital I/O card with 28 channels. The VIO4028 card:

    • Generates status signals for an operating status display on the hardware (DUT).

    • Reads the start button on the DUT.

    • Outputs the duty cycle signal for the PWM dimmable LED.

    • Reads in the 4 bits of the counter.

    • Generates the duty cycle signal for the fan speed adjustment.

    • Reads in the rotation speed of the fan (frequency measurement).

  • VIO1008: Voltage measurement card with 8 channels. The VIO1008 card:

    • Measures the voltage of the temperature sensor (as long as the I2C temperature sensor is not yet implemented).

    • Measures the voltage of the rotary control for the PWM dimmable LED.

  • VIO1804: Current measurement card with 4 channels. The VIO1804 card:

    • Measures the current consumption of the analog dimmable LED.

    • Measures the current consumption of the fan.

The dashboard of the sample configuration in your CANoe DE product consists of the following components:

  • A slider for the counter speed, the analog dimmable LED, and the fan speed.

  • A display for the position of the rotary control on the DUT.

  • Various displays for voltages, currents, duty cycle, etc.

Configuration Execution [▲ back]

The simulation runs in the desktop Configuration Execution. The following panels and windows are provided:

  • Dashboard
    This panel controls the VIO System and the Device Under Test (DUT). The buttons and setting options contained are described below.

  • Graphics
    The Graphics window displays important signals (system variables) of the simulation and is used for visual control of the simulation process.

  • Write
    The Write window displays important system messages, including status messages from the tests and compiler results (CAPL compiler).

The following describes how you can run the sample configuration.

Manual Test

Click Start to start the measurement in your CANoe DE product.

  • The CANoe Running signal is set to High to signal to the DUT that the connected CANoe DE product is running.

    • The traffic light on the TFT display changes from CANoe halted, DUT halted (red) to CANoe running, DUT halted (yellow).

    • Observe the CANoeRunning signal in the Trace window.

  • To start the DUT, click on the [DUT Start] button on the sample hardware.

    • The traffic light on the TFT display changes from DUT halted (yellow) to DUT running (green).

    • Observe the DutRunning signal in the Trace window.

  • Change the 4-bit Counter Speed Control slider in the dashboard of your CANoe DE product.

    • The VIO System outputs a variable voltage that is used in the DUT to set the speed of the counter.

    • Observe the 4 LEDs of the counter as well as the Counter Value and the Counter Freq. (Hz) displays.

    • For the frequency, the VIO System evaluates the changes to the least significant bit of the counter.

  • Change the Analog Dimmable LED slider in the dashboard of your CANoe DE product.

    • The VIO System outputs a variable voltage that directly controls an LED.

    • Observe the Analog Dimm. LED as well as the Analog LED Voltage and Analog LED Current displays.

  • Change the Fan Speed Control slider in the dashboard of your CANoe DE product.

    • The VIO System outputs a Fan Ctrl Duty Cycle control signal, which is used in the DUT to set the speed of the fan.

    • Observe the fan and its running noise, the Fan Ctrl Duty Cycle control signal and the displayed speed Fan RPMs.

  • Change the rotary control on the hardware (DUT).

    • The DUT outputs a variable voltage that is converted into a PWM signal in the VIO System.

    • Observe the PWM Dimm. LED as well as the PWM Control Voltage and LED Duty Cycle displays.

    • The PWM Dimm. LED changes its brightness with different duty cycles.

  • Change the distance between your hand and the ultrasonic sensor Sens1 (HC-SR04).

    • Observe the distance displayed on the dashboard of your CANoe DE product.

  • Carefully touch the temperature sensor U2 (MCP98243) or U4 (TMP36GZ) on the sample hardware (DUT) to warm it up.

    • Observe the temperature displayed on the dashboard of your CANoe DE product.

System Variables

Your CANoe DE product automatically generates the necessary system variables from the description files stored on the VIO System.

To increase the readability of the CAPL code as well as Ardiomp code, easy-to-read signal names are used.

Example: Signal indicating that the DUT is running:

  • Signal name: VIO4028::Out::DOut_Ch02

  • System variable: VIO4028:Ch02::DutRunning
