VIO System Simulated

 

Valid for:  CANoe DE  •  CANoe:lite DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • CANoe DE: VIOSystemSimulated.cfg
  • CANoe:lite DE: VIOSystemSimulated.cfg

Simulation of a VIO System with simulated, simple hardware as a Device Under Test (DUT).

The sample configuration VIO System Simulated illustrates the simulation of a VIO System in interaction with simple I/O hardware. For this, the inputs of the hardware are stimulated and the outputs are measured.

The (simulated) hardware consists of the following components:

  • A 4-bit Counter with adjustable voltage to change its speed.

  • An LED that is dimmable via an analog voltage. The voltage is set via a virtual slider in your CANoe DE product (Analog Dimmable LED Voltage (V)).

  • An LED that is dimmable via the duty cycle of a Pwm signal. The duty cycle is set via a slider on the simulated hardware (PWM Dimm. LED Duty Cycle (%)).

The VIO System consists of the following I/O cards:

  • VIO1008: Voltage measurement card with 8 channels. The VIO1008 card measures the voltage of the Duty Cycle slider.

  • VIO1804: Current measurement card with 4 channels. The VIO1804 card measures the current of the analog dimmable LED.

  • VIO2004: Stimulation card with 4 channels. The VIO2004 card generates the analog LED voltage as well as the control voltage for the speed of the counter.

  • VIO4028: Digital I/O card with 28 channels. The VIOO4028 card generates status signals for an operating status display on the hardware. It also outputs the Pwm voltage for the PWM dimmable LED. The card also reads in the 4 bits of the counter and the status of the start button on the hardware.

The dashboard of the sample configuration in your CANoe DE product consists of the following components:

  • The Start button. A signal is derived here that displays to the hardware whether your CANoe DE product is running. In a simulation, the CANoe DE product must always be running, but in the real hardware test environment, a display may be useful, which is simulated here.

  • The sliders for counter speed and analog dimmable LED.

  • Various displays for voltages, currents, duty cycle etc.

Manual Test

Click Start to start the measurement in your CANoe DE product.

The real DUT on which this simulation is based has a display of the operating status of the CANoe DE product. For this purpose, it evaluates the static signal CANoeRunning, which is set to High by the VIO4028 digital card as soon as the CANoe DE product is running.

This start check can be reconstructed in the simulation. To do this, the CANoe DE product must be started virtually within the running simulation as follows:

  • Click on the [Virtual CANoe Start/Stop] button.

    • The DUT Status display changes from CANoe halted (red) to DUT halted (yellow).

    • Observe the CANoeRunning signal in the Trace window.

  • To start the DUT, click on the [DUT Start/Stop] button (the real DUT has a physical start button for this).

    • The DUT Status display changes from DUT halted (yellow) to DUT running (green).

    • Observe the DutRunning signal in the Trace window.

  • Change the Counter Frequency (Hz) slider.

    • The VIO System outputs a variable voltage that is used in the DUT to set the speed of the counter.

    • Observe the 4 LEDs of the 4 bit Counter as well as the Counter Value and Counter freq. (Hz).

    • For the frequency, the VIO System evaluates the changes to the least significant bit of the counter.

  • Change the Analog Dimmable LED Voltage (V) slider.

    • The VIO System outputs a variable voltage that directly controls an LED.

    • Observe the Analog Dimm. LED as well as the Analog LED Voltage and the Analog LED Current displays.

    • As there is no real LED in a simulation, the value pairs are taken from a value table.

    • The dimming of the LED is simulated in the panel using different colors.

  • Change the PWM Dimm. LED Duty Cycle (%) slider.

    • The DUT outputs a variable voltage that is converted into a PWM signal in the VIO System.

    • Observe the PWM Dimm. LED as well as the PWM Control Voltage and the LED Duty Cycle displays.

    • The PWM Dimm. LED changes its brightness with different Duty Cycles.

    • As there is no real LED in a simulation, the value pairs are taken from a value table.

    • The dimming of the LED is simulated in the panel by different colors.

Automatic Test With vTESTstudio

Click Start to start the measurement in your CANoe DE product.

  • In the vTESTstudioTestConfiguration panel, select the tests that you want to run automatically.

    • You should always select the Start CANoe and DUT and Stop CANoe and DUT tests to ensure a defined start and stop.

    • In the PWMDimmableLedCheck test, the PWM control voltage is stepped through and the resulting duty cycle is checked.

    • In the CounterFreqCheck test, the counter frequency is stepped through and the measured frequency is checked.

    • In the AnalogDimmableLedCheck test, the analog LED voltage is stepped through and the resulting current is checked.

  • To start the test sequence, click Start in the TestConfiguration window.

    • Observe how the tests are running.

    • Observe how the signals change in the Graphicswindow.

    • Adjust the selection of tests and observe the results.

  • Click Open Test Report to open a test report at the end of the test run.

Note

No vTESTstudio license is required to run the tests created with vTESTstudio. If you want to modify tests or define your own, you need a vTESTstudio license. For this, please contact your Vector sales representative.

Namespaces

To show the relationship to a real VIO System, namespaces similar to those of a real system are used in this simulation.

Examples:

  • Real VIO System, output signal of a VIO4028 card, channel 1:
    Signal name: VIO::VIO4028Out::DOut_Ch01

  • Simulated VIO System, output signal of a VIO4028 card, channel 1:
    Signal name: VIO_simulated::VIO4028Out::DOut_Ch01

System Variables

To increase the readability of the CAPL code, easy-to-read signal names are used.

Example: Signal indicating that the DUT is running:

  • Signal name: VIO4028::Out::DOut_Ch02

  • System variable: VIO4028:Ch02::DutRunning
