    Shows, how your CANoe DE product can be used to send client requests using the HTTP protocol.

Introduction

When the configuration is loaded, you can send HTTP requests to an API during the measurement to get the corresponding sunrise times and sunset times for latitude and longitude coordinates as response.

Note

In this sample configuration you are using the Sunset and sunrise times API. Its instructions for use must be observed. The functions described here are depending on the functioning of the API.

Desktops

This configuration contains the following desktops:

HTTP Requests — External

On this desktop you will find the Sunrise API Control panel to control the HTTP requests to be sent, two windows of the type Trace and a Write window.

Sunrise API Control

This panel can be used to send HTTP requests to the Sunset and sunrise times API. In the panel, enter the longitude and latitude coordinates for which you want to get sunrise times and sunset times.

Select whether the HTTP request should be sent in High Level or Low Level mode.

In High Level mode, the client Application::HighLevel::SunriseApiClient modeled in vCDL is used.

In Low Level mode the provided client _HTTP::Client is used. The JSON serialization and deserialization is done by the serialization service with the object Application::Datatypes::SunriseApi::Serializer.

To start, click the Execute Request button.

Trace

The Trace window named Trace High Level or the Trace window named Trace Low Level will then display the corresponding method call and the return triggered by it.

HTTP Requests - Local

On this desktop you will find a panel to control the HTTP requests to be sent, two windows of the type Trace and a Write window.

Local Server Add User

At the start of measurement, a local HTTP server is started on port 8123. By means of a POST request in the Local Server Add User panel, user data can be stored in the server here. All users must have a unique ID therefor.

For each query, you can use the list to decide whether it should be sent in High Level or Low Level mode.

To execute the request, click the Execute Request button.

Trace

Then appears in the Trace window named Trace High Level or in the Trace window named Trace Low Level the corresponding method call and the return triggered by it.

Local Server Get User

In this panel you can send a GET request to the server by entering the USER ID, which then returns the stored information. If there is no user with the requested ID, an error notification will be returned with the code 404.

Local Server Delete User

With a DELETE request in this panel you can delete users from the database. If there is no user, the request is answered with a code 404.

In High Level mode the client Application::HighLevel:: LocalClient is used, which was modeled in vCDL.

In Low Level mode, the provided client _HTTP::Client is used. JSON serialization and JSON deserialization is done here by the serialization service using the Application::Datatypes::Local::Serializer object.

Modeling

On this desktop you will find the windows Measurement Setup, Communication Setup and a Write window. Here you can see the communication model. For the modeling the language vCDL was used as data source. To control the simulation, scripts in the C# language are used. A local HTTP server written in Python is also used for the HTTP Requests - Local desktop.
