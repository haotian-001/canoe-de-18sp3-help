System Configuration (SOA)

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • System Configuration (SOA) – SOASystem.cfg
    Shows the simulation and testing of a service-oriented architecture (SOA) for vehicles.

The communication concept of the vehicle architecture under consideration contains the following distributed objects:

  • Mirror_Driver provides the AdjustMirror method as a provider, which can be called by the consumer Console.
  • LIDAR_Front and LIDAR_Rear are the providers of the LIDAR events to which the Drive_Master reacts as a consumer and which it processes further to create Collision_Info.
  • The navigation data is made available by Backend as a provider to Drive_Master as a consumer.
  • Finally, the Dashboard manages the information in the following fields as a provider, which the Drive_Master writes as a consumer: Collision_Info_Front, Collision_Info_Rear, Navigation_Info and Velocity_Info.
  • The speed is represented by the system variable Velocity, which is simulated sinusoidally in the Signal Generator.

Thus, the objects used each represent one of the basic types of members of distributed objects.

All communication participants are simulated in your CANoe DE product. The simulation occurs directly on the service level; underlying network layers are not simulated.

Introduction

The measurement can be started directly. The Simulation desktop contains panels for the visualization and interactive stimulation of the communication participants as well as an automated test module.

The Analysis desktop contains the Trace window, which displays the communication on the service level. The Setup contains the Communication Setup.

Simulation [▲ back]

The Dashboard panel visualizes the values received from the Dashboard communication participant for the vehicle surroundings and the navigation data.

The LIDAR panel enables the interactive stimulation of the detected objects in the front and rear vehicle surroundings. The type of object and a distance between 0 and 100 m can be selected.

The application panel Backend enables the interactive stimulation of the navigation data transmitted by the Backend. Sender destinations or distances can be entered in the Value table column and sent using the Apply button.

The method control in the Mirror panel allows you to set and send the values of the AdjustMirror function. The changed mirror position and the mirror heating are displayed in the same panel. Valid values for the function are also indicated on the panel.

The Test test module (or the test unit Test Coniguration 1) performs an automated test. Stimulation takes place here on the LIDAR_Front, LIDAR_Rear, Backend and Mirror communication participants with test data. A check is performed to determine whether this test data is correctly received by the Dashboard, or Console communication participant.

Analysis [▲ back]

The Trace window records the communication on the member level. Here, you can follow the method calls as well as transmitted events between the communication participants.

Setup [▲ back]

The Communion Setup contains all configuration settings for the given configuration. Configured there are, in particular, the used services and application models.
