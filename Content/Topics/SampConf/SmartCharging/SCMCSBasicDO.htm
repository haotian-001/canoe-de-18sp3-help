MCS Basic Simulation (SmartCharging)

 

Valid for:  CANoe DE

Click on the blue link to load the sample configuration in your CANoe DE product.

  • MCS Basic Simulation – MCSBasicDO.cfg
    Demonstrates the Megawatt Charging System (MCS) communication between an electric vehicle (EV) and a charging station (EVSE) according to ISO 15118-10, ISO 15118-20 and IEC 61851-23-3.

Introduction

The configuration represents the most important features of the Option SmartCharging for the MCS protocol by connecting a vehicle simulation with a charging station simulation. You will find the following application models in the Communication Setup:

  • EVSEsimDO.can: Supplies data for the charging station simulation

  • EVsimDO.can: Supplies data for the vehicle simulation

  • Monitor.can: Helps to display the data

Desktops

This configuration contains the following desktops:

MCS State Monitor Configuration

MCS [▲ back]

On this desktop, you can observe the communication of a single vehicle with the charging station. The behavior of the participants can be influenced via the corresponding panels. To start the communication, simply click on the Connect vehicle button. The simulation runs automatically until the end of the communication.

In the Trace window, the smart charging messages (SDP, V2G) are displayed in decoded form with complete data interpretation.

Note

When the battery is full, the vehicle automatically stops the session.

EV

This panel serves as a control and display element for the vehicle simulation module. The following controls are available:

Operator Control

Description

Connector button

Connects the vehicle to the charging station or disconnects it again. As soon as the button is clicked, the SCC protocol sequence begins.

Battery

Indicates the current charging state charge status of the battery. The battery status can be reset using the [Clear] button.

Link

Displays the link status between EV and EVSE.

Vehicle state

The current state within the (simplified) SCC state machine.

Reset / Pause

The vehicle simulation can be paused and reset using the two buttons. A restart is triggered by a double click on the [Connector button] button.

Stop Charging

End the protocol in the regular way, from the charge loop.

ReNegotiation

Initiates a re-negotiation process while the protocol is in the charge loop.

MeteringConfirmation

Initiates a MeteringConfirmation process while the protocol is in the charge loop (only possible in PnC mode).

Certificate installation

Initiates a certificate installation during the next protocol execution (only possible in PnC mode).

ChargeDischarge

Controls the (dis)charge behavior during the loading loop. Discharging is only possible when using a BPT mode. The vehicle can only influence this behavior when the control mode called scheduled is used.

Control mode

Control mode used in some V2G messages that defines which side controls the charge/discharge behavior.

Schema

Use this switch to select the protocol version to be used for the connection. The protocol version is adopted as soon as the SupportedAppProtocol message is sent.

Charge Loop Interval

Controls for changing the loading loop interval, which defines the interval of successive requests within the charge loop.

Tab DC

Access to values that are relevant for the DC charging process: the charge mode (extended) as well as the voltage values of the requests.

Tab Limits

Electrical limits that are used in various messages.

EVSE

This panel serves as an operator control and display element for the chargeing station simulation module. The following operating elements are available:

Operator Control

Description

SessionID

Displays to the session ID of the active connection.

Link

Displays the link state between EV and EVSE.

Return FAILED

Instructs the charge point to label the next response message as an error.

Abort Conn.

Abortsthe connection immediately without notifying the vehicle.

Reset / Pause

The two buttons can be used to stop and reset the charging station simulation. A restart is triggered by a double click on the [Connector button] button.

ChargeDischarge

Controls the (dis)charge behavior during the charge loop. Discharging is only possible when using a BPT mode. The vehicle can only influence this behavior when the control mode called scheduled is used.

Stop Charging

Sends a stop request; the exact functionality depends on the selected scheme.

ReNegotiation

Initiates a re-negotiation process while the protocol is in the charge loop.

MeteringConfirmation

Initiates a MeteringConfirmation process while the protocol is in the charge loop.

Payment Options

Determines which payment options are offered by the EVSE.

EVSE Notification

Displays the last EVSENotification sent.

Tabs DC, DC_Limit

These influence the power, voltage and current limit values of the charging station, which are sent within several messages.

Protocol

In this panel, you can follow the ISO 15118-20 protocol sequence and configure the general behavior. When messages are received, the current protocol status is highlighted in the sequence view and the return value of the charging station is displayed. For simplicity, not every displayed status corresponds to a single message.

The following controls are available:

Operator Control

Description

Protocol bar

Indicates the current state of the protocol and the response codes sent by the EVSE.

Processing Time

If this option is activated, the charge point simulates a processing time for each message pair that contains the EVSEProcessing flag and causes a message loop. If this option is not activated, the simulation continues directly with the next message pair.

TLS

Activates the use of TLS the next time a connection is established. The configuration is carried out automatically over the supplied security profile.

PnC Active

Here you can switch between PnC and EIM mode. Activate the Active check box for PnC mode.

BPT Active

Switches between DC mode and DC-BPT mode. The charging mode is adopted as soon as the message ChargeParameterDiscovery is sent. It cannot be changed for the protocol session afterwards.

IEC61851-23-3

This panel is used for operating and displaying the hardware connection.

State Monitor [▲ back]

The MCS IEC61851-23-3 panel is used to display all voltages that have been sent or received by the hardware.
