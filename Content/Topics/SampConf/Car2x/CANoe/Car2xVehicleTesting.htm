Car2x Vehicle Testing – Emergency / Prioritized / Recovery

 

Valid for:  CANoe DE

Note

Click on the blue link to load the sample configuration in your CANoe DE product.

Car2x Vehicle Testing – Emergency / Prioritized / Recovery - Car2xEmergencyVehicleTest.cfg

Enables testing of the HLN-ERVI/HLN-EPVA use cases for a special vehicle (Emergency/Prioritized/Recovery).

Introduction

At the start of measurement, the system switches to the desktop Test. In the Car2x Station Manager, the station to be tested can be assigned to the DuT node in order to analyze the messages and test them with the test unit.

Desktops

This configuration contains several desktops.

Setup

Contains the Simulation Setup, the Measurement Setup and the Scenario Manager.

A Trace window and an inactive logging block are integrated in the Measurement Setup.

The database is integrated in the Simulation Setup. This contains the ASNV modules for the EU application messages. The DuT node is used for the station to be tested. The TesterNode node contains CAPL code for checking the messages (CAM and DENM). To analyze logging files, the replay block can be activated and the file can be replayed.

The Scenario Manager contains a scenario with an emergency vehicle that can be used for testing purposes to familiarize yourself with the configuration.

Test

Various analysis windows, a Write window, the test configuration and panels are displayed on this desktop:

  • The Map window visualizes the ITS stations and DENM events.

  • The Trace window displays the data traffic.

  • The Data windows display various values of the messages (CAM, DENM).

  • The Write window displays information, warnings and errors.

  • Car2x Station Manager to assign the station under test to the DuT node.

  • Panel Visualization for visual display and analysis of some message values.

  • Panel Test Control to start and stop the TestUnit, it displays the individual test steps.

If a DENM with CauseCode 95 or 15 is received, the sender of this message is automatically assigned to the DuT database node in the Station Manager. If desired, the node can also be assigned manually.

In the test configuration, the test unit is shown with its test sequence. In addition, the test unit can be opened in vTESTstudio or the vTESTstudio Viewer via the Edit Test Unit button. The programming of the test cases in CAPL can be traced here.

The test report can be viewed via the Open test report button.

The TestUnit can be controlled via the Test Control panel. It also displays the current test step. The test is stopped if one of the following conditions is met:

  • Stop Test button pressed

  • Key s pressed

  • DuT changes CauseCode (from in intervention to at location and vice versa)

Protocol Analyzer

This desktop contains a Car2x Protocol Analyzer and a Trace window.

Certificate Explorer

The Car2x Certificate Explorer is displayed in this desktop. This gives you a quick overview of the certificates used and their content.
