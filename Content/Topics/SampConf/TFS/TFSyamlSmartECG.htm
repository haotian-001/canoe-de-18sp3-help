    Shows how you can test an algorithm for determining heart rates from an ECG data set interactively or automatically with a test unit in the format vtestunit.yaml.

To open the test unit from your CANoe DE product, we recommend that you configure the editor for test units in the format vtestunit.yaml to Visual Studio Code. In addition, the extensions Vector CAPL, Vector Simulation and Test Environment and Vector Test Unit should be installed in Visual Studio Code.

Introduction

The software to be tested is available as a C application. The software receives ECG readings and uses them to determine the heart rate. It also checks whether the heart rate is in a critical range. Communication between your CANoe DE product and the software to be tested takes place via SIL Kit. This setup is illustrated on the Setup panel in the Exploratory Testing desktop. The interfaces of the system are modeled as distributed objects.

The C application can be built either with the help of the Visual Studio Solution on Windows or with CMake for Windows or Linux. To do this, open the folder via the Open SUT Folder panel on the Configuration desktop. The Visual Studio solution is located in the folder Application_Build. The executable created is located in the subfolder Application_Build\bin.

First start the measurement in your CANoe DE product and then the application via the file system.

You can influence the simulated heart activity via automated tests or the Start and Stop Sending Sample Data to SUT panel on the desktop Exploratory Testing.

This sample configuration contains data from the MIT-BIH Arrhythmia Database, which is provided under the ODC Attribution License v1.0. The MIT-BIH Arrhythmia Database was built as part of the following publication: Moody GB, Marker RG. The impact of the MIT-BIH Arrhythmia Database. IEEE Eng in Med and Biol 20(3):45-50 (May-June 2001). (PMID: 11446209).

The database is provided by PhysioNet: Goldberger, A., Amaral, L., Glass, L., Hausdorff, J., Ivanov, P. C., Marker, R., ... & Stanley, H. E. (2000). PhysioBank, PhysioToolkit, and PhysioNet: Components of a new research resource for complex physiologic signals. Circulation [Online]. 101 (23), pp. e215-e220.

Exploratory Testing [▲ back]

The Start and Stop Sending Sample Data to SUT panel allows you to manually change the simulation of cardiac activity. The START ECG Curve Simulation From Patient Data and STOP ECG Curve Simulation From Patient Data buttons start and stop the simulation of cardiac activity. The selection in Type of ECG Signal influences whether the simulation should simulate irregularities (With_Arrhythmia) or not (Without_Arrhythmia). The Simulate Emergency and Stop Emergency buttons can be used to simulate a critical state. The Current Heart Rate and Averaged Heart Rate displays show the current and averaged heart rate calculated by the software under test. The Heart Rate Alarm display shows whether the software under test has detected a critical condition.

The Graphics window visualizes the simulated heart activity as well as the values determined by the software under test as graphs.

The Trace window displays events and value transfers between the environment simulated in your CANoe DE product and the software under test.

Automated Tests [▲ back]

The test configuration SmartECG Test Configuration contains the test unit SmartECG, which was defined with the format vtestunit.yaml. The test cases are displayed in the execution dialog and can be executed via the Start button.

During execution, you can observe all actions in the Trace window Test Trace. See desktop Exploratory Testing.

The test cases are displayed in the execution dialog. You can open the details on the implementation of the tests via the Edit test unit button. To facilitate editing, it is recommended to install Visual Studio Code and configure it as described above.

Configuration [▲ back]

The Communication Setup contains all configuration settings. Among other things, the interfaces of the software to be tested are described here by distributed objects in the file SmartECG.vCDL.

The following files are also used to simulate cardiac activity:

  • SimulationControl.vCDL defines distributed objects that can be used to control the simulation.
  • InputMeasuredECG.CS implements the simulation of the measured values.
  • PanelHelper.can controls the panel Start and Stop Sending Sample Data to SUT controls on the desktop Exploratory Testing.
