Direct Access to Values from System Variables

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

You can read and change system variables without using the functions SysGetVariableInt/SysGetVariableFloat and SysSetVariableInt/SysSetVariableFloat.

For this purpose, write @Namespace::Variable.

Example

intValue = @Namespace1::Parameter2;
@Debug::MotorValues::EngineSpeed = $EngineSpeed;

You can also omit namespaces (from left to right) if you defined them in the variables section with a using namespace <namespace> command.

Example

// System variables:
// Namespace1::Namespace2::Var1
// Namespace1::Namespace3::Var2

variables
{
using namespace Namespace1;
using namespace Namespace1::Namespace2;
}

on key '1'
{
@Var1 = 1;
@Namespace3::Var2 = 2;
}

  • Access to symbolic values of a value table (in CAPL)
    The assignment follows the following schema: (Syntax) @sysvar::<SVPath> = sysvar::<SVPath>::<Description>.

    Example

    @sysvar::Geberradadaption::Active_Diagnostic_Session = sysvar::Geberradadaption::Active_Diagnostic_Session::Extended;

  • Access to single array elements

    Example

    intValue = @Namespace1::ParameterArray[2];

  • Access to single elements of a system variable of struct or generic array type
    Add the element to the name of the variable.

    Example

    @XCP::ECU_2::KL2.Curve2[0] = 1.3;

    If a conversion formula is defined for the element, the direct access returns the raw value per default. You can also read out the physical value by adding .phys.

    Nevertheless by doing so you don't have access to complete strings or arrays.

Note for CANoe DE, CANoe4SW DE

System variables defined in the XML test module (not in your CANoe DE product) cannot be accessed in your CANoe DE product using @ method, because the object is not existent in your CANoe DE product.

In comparison: a system variable defined in the XML test module cannot be selected in the Symbol Explorer of your CANoe DE product.

Note regarding versions older than 7.0

With versions before 7.0 you have to specify the data type of the variable by putting sysvarInt:: or sysvarFloat:: in front.
Access to arrays is not possible. The existence of system variables is not checked until runtime.

Versions before 6.0 do not allow direct access to system variables.
