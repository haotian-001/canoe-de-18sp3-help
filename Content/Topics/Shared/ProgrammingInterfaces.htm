Programming Interfaces (APIs)

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

Various programming interfaces are available to extend the functionality.

Component

Description

CAPL

The Communication Access Programming Language (CAPL) allows programming of network node models as well as special evaluation programs for individual applications.

C Library

The C library feature (CCL for short) enables you to write application code in the C programming language and execute it as part of a measurement.

A simple C API for accessing services of your CANoe DE product is available in your application code. This API enables the sending and receiving of CAN and LIN frames, provides timer services and enables access to the system variables of your CANoe DE product.

A development environment for C is not included in the scope of delivery.

COM

With the help of the COM Interface you can control your CANoe DE product from external programs.

Besides applications, scripts also come into consideration as external programs.

Certainly the most well-known script and programming languages available to you are Python, PowerShell, C# and Visual Basic.

.NET

You can use .NET programming languages in various places in your CANoe DE product.

A special API for .NET programming is offered that specifically extends the languages (i.e. creates Embedded Domain Specific Languages). C# (programming language recommended by Vector) and Visual Basic .NET are supported as programming languages.

Python

In your CANoe DE product you can use Python scripts for programming application models.

For security reasons, no interpreter is shipped. Only a special API extension is provided and thus an Embedded Domain Specific Language.

You can also use Python outside your CANoe DE product to control it via the integrated COM server.

vCDL

vCDL allows the communication concept to be extended by a tool for the simple and text-based configuration of application layer objects.
