VIO System Variable

 

Valid for:  CANoe DE

When you configure one or more VIO systems, your CANoe DE product also creates a variable VIO_System_Event in the system variable VIO. This is suitable for programmatic error analysis and contains several signals that describe the error that occurred. In the event of an error, only the applicable signals are set; all others remain empty. The signals are for read-only purposes; manually overwriting the signals has no effect. The following signals can be set by your CANoe DE product:

The severity of the error message if the triggered event is an error. It can take the values Info (0), Warning (1), Error (2) and CriticalError (3). Critical errors differ from normal errors in that a critical error automatically aborts the measurement.

Describes the type of triggered event. The types NoEvent (0), CANoeError (1), ViolationOfValueRange (2), XCPEvent (3) or DeviceEvent (4) are possible.

The system variable VIO_System_Event indicates the IP address (127.0.0.1) and the card slot (5a) so that the source of the error can be clearly localized for VIO Systems with multiple stimulation cards of the same type. The system variable also shows that the error source is channel 0. The generated EventCode (8589934596or 0x200000004) can be used to provide clear programmatic error handling.

In the event of a value range violation (see EventType ViolationOfValueRange), the system variable displays the value name (PwmFrequencyOut_Ch01), unlike in the first example, because the value range violation occurred when an invalid frequency was set in your CANoe DE product. In this case, the invalid frequency (26000) can also be found in the error description. Since there is only one error mode for value range violations, no EventCode is generated.
