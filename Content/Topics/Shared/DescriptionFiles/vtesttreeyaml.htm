In aâ¯vtesttree.yaml a test execution tree is described textually. Note that vtesttree.yaml is not a filename but a file extension, e.g., filename.vtesttree.yaml.

In order to execute the tests, the vtesttree.yaml must be referenced in a vtestunit.yaml.

The format used is YAML. You can edit this format with any text editor, but a text editor with YAML support is recommended.

Structure of vtesttree.yaml

You can find a complete scheme structure here: vtesttree.yaml schema

version: 2.2.0
test-tree:
  - ...

Test Execution Tree

The test execution tree determines in which order test cases and test sequences are executed. The tests are executed from top to bottom in the order they have been entered in the YAML file. Currently, test cases and test sequences programmed in CAPL, Python or C# can be executed.

The corresponding test cases and test sequences have to be defined within source code files that are referenced by the same vtestunit.yaml that references the vtesttree.yaml. In addition, they must be marked as exported.

Test fixtures can be used to further structure the test execution tree.

test-tree:
  - capl-test-case: MyTestCase
    ...
  - capl-test-sequence: MyTestSequence
    ...
  - python-test-case: my_python_test_case
    ...
  - dotnet-test-case: MyDotnetTestCase
    ...
  - test-fixture: MyTestFixture
    capl-preparation: MyPreparation
    elements:
      - capl-test-case: ...

Definition of the CAPL functions for this example:

export testcase MyTestCase() {
...
}
export testsequence MyTestSequence() {
...
}
export testfunction MyPreparation() {

...

}

Variant Properties

To link the execution of a test fixture, a test case or a test sequence to the value of a variant property, you must first define it in the vtestunit.yaml file. The variant-dependencies node can then be used to formulate conditions as to when or for which value of the variant property an element should be executed. If the node is not specified, the execution is independent of the value of the variant property.

Conditions have the form <variant property> <comparison operator> <value>. The comparison operators == (equality) and != (inequality) are permitted. Several conditions can be linked with the operator || (logical or).

To execute the CAPL test case MyTestCase from the example above only for the values B and C of the variant property ECU_Model, you can define the call in the test execution tree as follows:

test-tree:
  - capl-test-case: MyTestCase
    variant-dependencies: ECU_Model == B || ECU_Model == C
    ...

Definition of the variant property in vtestunit.yaml for this example:

variant-properties:
  - name: ECU_Model
    value-table: [A, B, C]
    ...

Tool Recommendation

Visual Studio Code with YAML Extension from Red Hat.

The vtesttree.yaml schema should be obtained automatically from the schema store. If this is not possible, for example because there is no Internet or there are proxy problems, you can also configure the schema manually in the Visual Studio Code settings.

{
  "yaml.schemas": {
    "file://<Tool_Installation_Directory>/Exec64/schemas/vtesttree.yaml-schemas/vtesttree-schema.json"
: ["*.vtesttree.yaml"]
  },
