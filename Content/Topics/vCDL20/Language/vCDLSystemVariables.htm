System Variables

 

Valid for:  CANoe DE  •  CANoe:lite DE  •  CANoe4SW DE  •  CANoe4SW:lite DE

System variables as such are not a new feature, but they can be defined in vCDL starting with version 2.0.

Example

sysvar int32 IntegerSysVar;

[UsedForAnalysisResults]
sysvar float FloatSysVar = 5.6;

[ReadOnly, Transient]
sysvar string StringSysVar = "Initial Value";

[SysVarType, TimeBetweenValues=56ms]
sysvar array<int32, 10> ArraySysVar;

[SysVarType, SysVarBytesLength=10]
sysvar bytes BytesSysVar;

struct MyStruct
{
  int    Member1;
  float  Member2;
  string Member3;
}

sysvar MyStruct StructSysVar = { 4, 5.6, "Test" };

System variable definitions have the same structure as constants, but obviously with a different keyword and different attributes. Also, the initial value is optional.

Attributes

Name

Type

Default

Semantics

ReadOnly

bool

false

Whether the system variable is read-only.

TimeBetweenValues

nullable<time>

null

Time between consecutive values in nano seconds If the system variable is a value sequence or null otherwise.

Transient

bool

false

Whether the system variable is transient.

UsedForAnalysisResults

bool

false

Whether the system variable is used only for analysis which means changes are not transmitted to RT.

There are more attributes which technically belong to the system variable's data type, but not the system variable itself. They tell your CANoe DE product to use special data types for bytes and array<> which are performance optimized for system variables and thus cannot be used in any other context.

Name

Type

Default

Semantics

SysVarType

bool

false

Enables the usage of the optimized data type tailored to system variables. Only usable on data types bytes and array<>. The latter requires the element type to be any of int32, int64, uint32, uint64, float, or double.

SysVarBytesLength

nullable<uint>

null

Optional length for system variable optimized bytes type. If set, the bytes type has a fixed layout.

Syntax

system-variable:

attributes sysvar data-type identifier ( = expression )?
