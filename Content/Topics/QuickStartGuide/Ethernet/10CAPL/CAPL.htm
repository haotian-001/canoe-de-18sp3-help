Extend Simulation

 

Valid for:  CANoe DE

You have already added the CANoeILNL_AUTOSAR_Eth.dll interaction layer to all simulation nodes via the VMODULE files (see Simulation). This interaction layer starts to simulate the ADAS ECU as well as it is described in the database describing the network or only the single node (ECU extract). In addition, the interaction layer offers CAPL functions for directly influencing the simulation.

As an example, let's write CAPL code to cancel a SOME/IP service consumed by ADAS. The CAPL functions that cause the SOME/IP consumer to subscribe or unsubscribe to a particular SOME/IP service are AREthSDSubscribeEventGroup and AREthSDDesubscribeEventGroup.

To switch the subscription on and off during the measurement, you have the following options:

  • CAPL event procedure on key '<Char>'
  • Panel with a switch to which a system variable is assigned

The first option is quick to implement. But without a good documentation of your configuration it is hard to recognize the possibility of influencing the simulation.
The second possibility is not much more complex and self-explanatory.

This tour explains the second option. To do this, you must create a system variable, expand the panel you have already created and fill the ADAS.can file with CAPL code.

Create a System Variable

To model the functional bus behavior of network nodes, CANoe DE provides you system variables that are used to describe events and states of the system environment (external pressure, temperature, switch positions, ...). You can both observe and change these states (the values of the system variables) on freely configurable panels.

To work with system variables, use the event procedure type on sysvar ("react to system variable change") in CAPL. The CAPL functions SysGetVariableLong and SysSetVariableLong, for example, are used to read and write system variable values.

For our tour, you need a system variable to switch the consumption of a service on or off. A simple integer is sufficient.
Give the variable the meaningful name ADAS::sysService2001Evg1. ADAS is used as a namespace. This makes it easy to understand which network node the system variable influences.

Step by Step Procedure

  1. Open the System Variables Configuration dialog via the Environment ribbon tab.

  2. Open the New System Variable dialog via in the toolbar.

  3. Enter the following values:

    Namespace: ADAS

    Name: sysService2001Evg1

    Data type: Integer (32-bit signed)

    Location: Configuration

  4. Close the dialog with [OK].

Insert a CAPL Node

CAPL is an event based programming language. Each CAPL program consists of event procedures, with which you can react to external events (e.g. occurrence of specific messages on the CAN bus or activation of keys on the computer keyboard). The CAPL Browser is described in detail in the help of CANoe DE. With its sub windows ("Panes") it allows you to create and edit CAPL programs quickly and easily.

In principle, you can also use your own text editor to create CAPL programs. CAPL programs are normal ASCII files with the default name extension *.can, which must be compiled before the start of measurement using the compiler provided with the CANoe DE product.

CAPL programs can be added to network nodes in the Simulation Setup or to hotspots in the Measurement Setup. There are two important differences:

  • CAPL programs in the simulation setup run with higher priority. Timely reactions can only come from here.
  • CAPL programs in the measurement setup cannot access the real network. Therefore, they cannot send any packets.

To add more logic to the simulated ADAS controller, you must add a CAPL program file to the ADAS network node.

Step by Step Procedure

  1. Switch to Configuration desktop.

  2. Select the ADAS node in the Simulation Setup.

  3. Open the Node Configuration dialog with the context menu command Configuration....

  4. Open the Open dialog with [File...] on the General page under Node Specification.

  5. Enter ADAS.can here and click on [Open].

    You have now created the ADAS.can file.

  6. Close the dialog with [OK]

Write a CAPL Program

You can now open the file in the CAPL Browser using the yellow pencil in the node symbol in the Simulation Setup and enter the following CAPL code:

variables
{
  char eventGroup1[20] = "sif_2001::evg_1";
  long result;
  dword handle;
}

on sysvar sysvar::ADAS::sysService2001Evg1
{
  handle = AREthGetConsumedObjectHandle (eventGroup1);
  if (handle == 0)
  {
    // Error
    write ("Error: Service object handle not available");
    return;
  }

  if (@this) // Subscribe to service
  {
    result = AREthSDSubscribeEventGroup (handle, 0);
    if (result != 0) // only 0 is ok
    {
      write ("Error: Subscribe to service/eventgroup not successful");
      return;
    }
  }
  else // unsubscribe
  {
    result = AREthSDDesubscribeEventGroup (handle);
    if (result != 0) // only 0 is ok
    {
      write ("Error: Unsubscribe service failed!");
      return;
    }
  }
}

Check your CAPL program via Compile on the Home ribbon tab. This will display all syntax errors in the output window that you need to correct in order to be able to start a measurement at all.

Save the file and close the CAPL Browser.

Extend the Panel

Step by Step Procedure

  1. Open the Panel Designer via the Tools ribbon tab.

  2. Open the SIGNAL.xvp panel by double-clicking in the Project Documents window.

  3. Select the Check Box in the Toolbox on the right under Common and place it on the panel using drag-and-drop.

  4. Select the system variable ADAS::sysService2001Evg1 in the Symbol Explorer on the left and assign it to the Check Box using drag-and-drop.

    Switch to the system variables view with in the toolbar.

  5. Save the panel.

Start the Measurement

Activate the Check Box for sysService2001Evg1.

Start the measurement. The 2001 service with Event Group 1 is the Traffic Sign Detection of CAMF. ADAS is the only consumer for this service.

Check that the value of the SignTypes signal in the Data Window changes every time you select a different type in your panel.

Deactivate the Check Box for sysService2001Evg1 and try to change the value again. As you have now deregistered the consumer from the service, the CAMF node no longer sends any events and the selected value is no longer displayed in the data window. In the panel, you can see from the symbol on the panel that the new signal has not yet been sent.
