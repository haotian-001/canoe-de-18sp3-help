Set Up the Hardware

 

Valid for:  CANoe DE

To connect the PC and thus CANoe DE to an Automotive Ethernet, you need a VN interface for Ethernet. VN interfaces for Ethernet have numbers starting with 5, e.g. VN5430 or VN5611. For the following examples, you need an interface with at least two ports and a loopback cable (Vector Ethernet cable with adapter connected to a loop). The VN interface must support CANoe DE simulations.

A VN5640 is used for the tour.

Connect the Loop Cable to Your VN Interface

First connect a loopback cable to your VN interface to connect two ports directly to each other. If the cable uses both ports on one IX connection, you can connect ports 1 and 2. If you have a cable with two ends and two IX connectors, you can connect ports 1 and 3. In this tour we will connect port 1 and 2.

Connect the VN Interface to Your Computer

Use a USB cable to connect the VN interface to your computer.

Note

Many Ethernet interfaces can also be connected via an Ethernet cable. However, this requires correct handling of the IP address. A license stored on the interface can only be found by CANoe DE if the connection is made via USB.

Configure Your Hardware

Step by Step Procedure

  1. Once you have connected the VN interface, you must configure it.

  2. To do this, open the Network Hardware Configuration dialog by clicking on Network Hardware on the Hardware ribbon tab.

  3. Select VN Ethernet hardware as hardware type.

  4. Click on [Driver...] to open the Vector Hardware Manager.

  5. Switch to the Configuration View.

    The configuration saved on the VN interface connected to the computer is displayed under Ethernet Network Configuration.

    In our example, no configuration exists yet and must first be created.

    For our other examples, we need two connections on two separate segments.

  6. Click on [Add network] and give the network the name Ethernet1. The network must have exactly the same name as the network in CANoe DE, as the connection between the networks on the VN interface and in CANoe DE is established automatically via the network name. If the names are different, you must connect the networks manually.

  7. Add a segment of the type Switch and assign it to your VN interface . For our tour, gPTP frames are not forwarded.

  8. Add a port with the name Port1 to the switch segment and select a suitable mode for it. For automotive Ethernet connections, one side must take the lead for synchronization. This port is referred to as the master. The other port must be of the slave type and in this step we select the Master.

  9. Add a second segment Swith2 with the same settings as Switch1.

  10. For Switch2, add a port with the name Port2 and select Slave mode.

    Note

    If you connect a real ECU or a real Ethernet switch, its operating mode is probably unknown. In this case, it makes sense to select the Master/Slave Auto Detect mode for both ports.

  11. Transfer the new configuration to the VN5640 by clicking on [Deploy] at the top right of the Vector Hardware Manager.

    You can also save the configuration in a file. This is useful if the interface is used repeatedly for different applications. To do this, click on the menu icon at the top left and select Save As....

    If you want to load a saved configuration onto an Ethernet VN device, click on the menu icon at the top left, select Open... and then click on [Deploy] again.

    The 100BASE-T1 and 1000BASE-T1 hardware layers require the same configuration steps. If you are using 10BASE-T1S as the physical layer, the hardware configuration is slightly different, see Appendix.

  12. Close the Vector Hardware Manager.

  13. Close the Network Hardware Configuration dialog with [OK].

Configure the Ports in CANoe DE

Automotive Ethernet is a fully switched network. Packets are transmitted over many point-to-point connections. Sometimes it is important to verify all parts of this journey. In many cases, however, listing the same information multiple times causes unnecessary overhead and leads to confusion.

Therefore, it makes sense to decide where frames should be displayed and where they should be hidden when setting up CANoe DE. By default, CANoe DE displays all packets from virtual ports and hides those from physical ports.

Ethernet packets that are sent via the real ports Port1 and Port2 are not displayed in the Trace window by default.

Therefore, activate both measurement ports via the Hardware ribbon tab in the Port Configuration dialog in order to see their Ethernet packets later.
