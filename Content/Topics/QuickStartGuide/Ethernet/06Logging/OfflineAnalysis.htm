Offline Analysis

 

Valid for:  CANoe DE

A common use case is the analysis of data traffic that was recorded at an earlier point in time. This is known as offline analysis and uses the offline mode of CANoe DE. A logging file can be analyzed in the same way as the data traffic during the measurement.

For our tour, we will now assign a database for interpreting the data and add an existing logging file to the configuration.

Assigning a Database

Of primary interest for the analysis of automotive networks is not only the bus or network-related information such as packets, transmission errors and packet flow, but also the user data of the packets. Signals such as speed, temperature and engine load, which are provided by individual control units and sent via the network. The necessary descriptions for signals in Ethernet packets are usually provided in an AUTOSAR database (ARXML) or as a Fibex database (XML) and can be displayed and interpreted after assignment in CANoe DE.

Step by Step Procedure

  1. Create the Databases subdirectory in your configuration directory.

  2. Copy the database from the sample configurations directory CANoe Sample Configurations\Ethernet\Analyse\EthernetBasic\DB\Network.arxml to this directory.
    The storage of the sample configurations depends on your installation. By default, they are installed in the following directory: C:\User\Public\Documents\Vector\CANoe\Sample Configurations <Version>

  3. Open the Simulation Setup (Configuration desktop or Simulation ribbon tab|Simulation Setup).
    In the System View of the Simulation Setup, you will see a tree view of the current configuration.

  4. Select Databases there and open the Open dialog with the context menu command Add....

  5. Select the above-mentioned database in this dialog.

  6. Click [OK] to accept the new database and display it in the System View.

You can now see the information contained in the database in the Symbol Explorer on the Environment ribbon tab.

Activating Offline Mode

Select offline mode on the Home ribbon tab or switch to offline mode by double-clicking on the Online/Offline symbol in the Measurement Setup. All configuration settings are transferred to offline mode.

The bus symbol is now deactivated in the Measurement Setup. Instead, the file symbol is activated as the data source.

Data received via the VN interface is now ignored and the CANoe DE simulation is inactive.

Selecting Data Source

Step by Step Procedure

  1. Create the Logs subdirectory in your configuration directory.

  2. Copy the logging file from the sample configurations directory CANoe Sample Configurations\Ethernet\Analyse\EthernetBasic\LOG\Ethernet.blf to this directory.

  3. Open the Offline Mode Window via the Offline File List context menu command of the file icon on the left in the Measurement Setup.

  4. Add the logging file either via in the toolbar or via the Add Offline Source... context menu command.

Playing Logging File

Start the measurement with <F9> or with on the Home|Messung|Start ribbon tab. The data is now read from the logging file and played back. You can use the same analysis windows for the evaluation as for a measurement with real data.

In addition, you can perform animated playback with a fixed number of events per second (<F8> or with on the Home|Measurement|Animate tab) or in manual steps (<F7> or with on the Home|Measurement|Step tab). For manual steps, you can enter either a time span or a fixed number of events in the field after the step length. Both options are repeated when you click on the [ Time Step] button.

Enter 2 s for the tour in the field after the step length. All events within the entered time period are now sent.
