Create a Logging File

 

Valid for:  CANoe DE

CANoe DE has extensive logging functions for data logging. In the Measurement Setup (Configuration desktop or Analysis ribbon tab|Measurement Setup) the logging branch is shown at the very bottom of the screen. You can easily recognize it by the file icon , that symbolizes the logging file. The logging file is filled with CAN data during the measurement.

Activate the Logging Branch

To log the data that arrives in the Measurement Setup to a file, first stop the measurement and activate the logging branch. Also remove the break that separates the Logging Block of a new configuration from the data source. You can do this by double clicking the break symbol or with its context menu command Remove Break.

Configure the Logging File

Use the context menu command Logging File Configuration... of the file icon on the right in the logging branch to open the configuration dialog.

You must make at least the following settings here:

  • Destination directory: Storage location of your logging file.

  • Destination file: Name of the logging file.
    The name can be defined together with a static and a dynamic part. For example, all files start with the same character string QuickStart_ and the dynamic part is put together and added when the measurement is started. In the example above, the dynamic part is the date and time of the start of the measurement.

  • File format: Format of the logging file.
    The BLF, MF4 and ASC formats are very suitable as logging formats for recording messages. BLF was developed for CANoe DE. MF4 and ASC contain the same information as BLF, whereby MF4 is an ASAM standard and ASC stores ASCII characters instead of binary data and takes up much more memory space. For Ethernet communication, PCAP and PCAPNG are also supported by CANoe DE. Both lack information, especially PCAP is not recommended.
    Recordings in binary format BLF require less storage space, but cannot be read in a normal text editor. The offline mode of the program offers you the same evaluation options for recordings in both formats.

CANoe DE supports all of the above formats in offline mode.

Configure the Trigger Conditions

By default, the entire measurement is written to a file. However, you can define trigger conditions to limit the recording to a specific time interval, e.g. start when the temperature is within a valid range, manual start of the recording, stop when the tests are complete.

Use the Configuration... context menu command of the logging block to open the Trigger Configuration dialog.

  • To record the entire measurement, you must select the Entire measurement mode in the Trigger Configuration dialog.
  • Single trigger starts the recording with a condition and runs either for a specific time or until the end of the measurement.
  • Toggle trigger defines the recording of one or more time intervals.

Analysis of a Logging File

The most complete way to analyze a logging file is the offline mode of CANoe DE. Add the logging file to your offline folder list, switch CANoe DE to offline mode and analyze the data with all analysis windows in the Measurement Setup.

Sometimes this is not necessary and it is sufficient to open the logging file with the Trace Window. Then drag a logging file with the mouse pointer from Windows Explorer into your Trace Window in CANoe DE. Or right-click in the Trace Window and select Import from the context menu. A logging file can also be imported into a Graphic Window instead.
