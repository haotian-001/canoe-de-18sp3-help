Replay a Logging File

 

Valid for:  CANoe DE

Sometimes tests can be carried out with little effort by replaying a logging file. The logging file contains events that were recorded during a measurement. Playing a logging file is much less time-consuming than creating a simulation of the rest of the network.

Note

Replay may not be possible for higher-level protocols or security-relevant communication. For example, AUTOSAR PDUs sent via the TCP protocol instead of UDP can be sent but not received. The handshake required by TCP does not work with recorded data.

More details on this topic can be found in the application note AN-IND-1-025_Ethernet_SOMEIP_Replay.pdf in our Down Load Center.

Simple replay may not be possible with higher-level protocols or security-relevant communication.

You can use the Replay Block to feed recorded packets into a real network.

In our tour, you will first create a logging file with filtered data from your simulation.

Creating a Logging File

To do this, you will run the simulation and change the VehicleSpeed signal over time and write the AUTOSAR PDUs from VGW to a logging file.

Configure the Logging Block

Activate the logging branch by double-clicking on the interruption symbol in the Measurement Setup.

Configure the entire measurement for the logging block in the Trigger Configuration dialog.

For the logging file, select the name VehicleSpeed_data_{MeasurementStart}.blf in the Logging File Configuration dialog.

Insert a IP Filter

To prevent unwanted data from being logged, you must insert a filter before the Logging Block.

Step by Step Procedure

  1. Select the hotspot before the Logging Block in the Measurement Setup.

  2. Insert an IP Event Filter using the context menu command Insert Event Filter|IP.

    A blue block labeled IFB appears behind the hotspot.

  3. Double-click on the IFB block to open the IP Filter.

  4. Switch to the Port page and configure the filter as follows:

    Type: Pass Filter ()

    Add a port via the context menu and configure it as follows:

    Channel: Eth 1

    Ports: VGW

    Direction: RX, as only packets coming from VGW should be recorded.

    Destination Address of node IC: 192.168.3.4

    Destination Port at node IC: 40001 ( = 9C41h)

    Depending on the setting for the number representation, you must enter a decimal or hexadecimal number here. You can see the setting on the Home|Display tab.

  5. Close the window.

Log the Data

Step by Step Procedure

  1. Start the measurement.

  2. Use the slider on your SIGNAL panel to change the signal values of the VehicleSpeed signal several times.

  3. Observe the signal value changes in the Graphic Window.

  4. Stop the measurement.

  5. Select the logging file in the Measurement Setup.

  6. Open the target directory via the context menu and check whether your logging file has been created.

  7. Deactivate the logging branch in the Measurement Setup by double-clicking on the hotspot in front of the IFB block.

Insert a Replay Block Ethernet

A Replay Block Ethernet is integrated in the Simulation Setup in the same way as the other network nodes.

Step by Step Procedure

  1. Insert a Replay Block Ethernet in the Simulation Setup via the Replay Block Ethernet context menu command of the network line.

  2. Open the Replay Configuration dialog with a double-click.

  3. Select your recorded logging file as the source file and accept the default settings in the dialog with [OK].

Replay the Logging File

Simultaneous use of simulation and playback is not impossible, but causes additional complexity. Therefore, deactivate all nodes in the Simulation Setup except for the Replay Block.

To do this, select a node and deactivate Block active in the context menu.

If you still have your VN interface connected, you must check the port configuration.

To do this, open the port configuration via the Hardware ribbon tab.

Assign Switch2 to the Replay Block under Simulation Ports.

Start the measurement.

Observe the signal curve of the VehicleSpeed signal in the Graphic Window on the Measurement desktop. The curve should correspond to the recorded curve.
