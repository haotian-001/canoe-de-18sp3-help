Transmit Data Without Database

 

Valid for:  CANoe DE

There are several ways to send an Ethernet packet to your network. These three are presented in this introduction:

  • Ethernet Packet Builder
  • Ethernet Interactive Generator
  • CAPL (See Extending a Simulation)

Ethernet Packet Builder

The Ethernet Packet Builder is the easiest way to prepare an Ethernet packet and send it to a network.

Insert an Ethernet Packet Builder in the Simulation Setup

Step by Step Procedure

  1. Insert an Ethernet Packet Builder in the Simulation Setup via the context menu command Insert Ethernet Packet Builder of the network line.

  2. Double-click to open the Ethernet Packet Builder.

  3. Click [Configuration...].

    The Ethernet Packet Builder uses default values for several address components. Enter the following settings in the open Configuration dialog and accept them with [OK].

  4. Add an Eth II, TCP and ICMPv4 packet via the selection list under [Add Packet]. The details of a selected packet are displayed under Packet Information. You can change the payload and all other information of each packet.

  5. Select Port1 for the packets to send them:

    Leave the Ethernet Packet Builder open.

You can also drag and drop any packet from the Trace Window into the Packet Builder. This makes it easy to resend a specific information without a lot of typing.

Saver the Configuration

Save your configuration before you start the measurement.

You can save the configuration either via the quick access toolbar or via the s File ribbon tab.

Start the Measurement

Start the measurement with <F9> or with on the Home ribbon tab.

Now select the generated packets one after the other and click on [Send] for each packet.

Outpu in the Trace Window

Switch to the desktop Trace.

In the Trace Window that is already open, you can see that the packets have been sent.

The packets are sent via the port of the VN interface that you have configured in the Ethernet Packet Builder (direction attribute is Tx). They are received via the second port (direction attribute Rx).

Each packet is displayed in a separate line.

If you wish to get detailed information on a line, select the line and open the Detail view with in the toolbar of the Trace Window.

Ethernet Interactive Generator

The Ethernet IG supports some of the requirements for higher-level protocols. It is useful for quickly establishing communication for the TCP protocol.

The Ethernet IG can send packets periodically.

Insert an Ethernet IG in the Simulation Setup

Step by Step Procedure

  1. Insert an Ethernet IG in the Simulation Setup via the context menu command Insert Ethernet Interactive Generator of the network line.

  2. Double-click to open the Ethernet IG.

  3. Switch to the Configuration page and make the following settings.

    You can edit the values by double-clicking on the respective cell.

    Leave the Ethernet IG open.

Unlike the Ethernet Packet Builder, the Ethernet IG knows the protocol requirements of UDP and TCP. If you configure it to send a TCP message, it will first initiate the TCP handshake with the target socket. If the target socket is not reachable, the Ethernet IG will not send a single TCP message or will abort after the first handshake request.

Configure the Ports for the Ethernet IG

Before you start the measurement and send packets with the Ethernet IG, you must assign the virtual port to one of the two switch segments of the VN interface. CANoe DE can make this assignment automatically if there is only one option. In this tour, you must configure it manually, as there are two options.

Step by Step Procedure

  1. Open the Port Configuration dialog via the Hardware ribbon tab.

  2. Select Switch1 for the EthernetIG simulation port.

Add Packets in the Ethernet IG and Send Them

Step by Step Procedure

  1. Switch to the Messages page in the Ethernet IG.

  2. Select a UDP message via [New].

  3. Start the measurement.

  4. Click on [Now] in the message line of the Ethernet IG.

    Red entries are displayed in the Trace Window. Red does not mean that there is an error. Red is the standard color for Ethernet ARP Frames (Address Resolution Protocol). However, you will not see the UDP message that you have just sent.

    When using higher level protocols such as UDP or TCP, only the IP address and port number are used as address information. MAC addresses are usually not known at this level of the OSI layer. And in fact, you could not specify a destination MAC address for the UDP message.When a UDP message is sent, the MAC address is queried via the Ethernet IG based on the IP. If the requested IP is found, a response is returned and the UDP message can be sent. If no response is received, the UDP message cannot be transmitted.

Evaluate Data

Column Configuration in the Trace Window

You have already become familiar with the Trace Window. It displays each event in one or more separate lines. The information about the event is displayed in columns. There are various options for configuring the display of the information. Here are two examples:

  • Select one of the predefined column settings via the Trace Window toolbar:

  • You can add or remove columns via the Column Configuration in the Trace Configuration dialog ( in the toolbar)

  • You can add columns via the context menu of the column headers via the Field Chooser or remove them by dragging them with the mouse.

Statistics Window

You can open the Bus Statistics Window via the Analysis ribbon tab. The window provides an overview of the data traffic for each network configured in CANoe DE. It shows the bus traffic, the transfer rates for packets and bits. The link status is also displayed for Ethernet networks.
