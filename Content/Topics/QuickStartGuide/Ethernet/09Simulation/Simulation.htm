Simulation

 

Valid for:  CANoe DE

This chapter describes the procedure for simulating an Ethernet network.

Creating a Simulation

The database already assigned is used for the simulation.

Now select the online mode on the Home ribbon tab.

Add Network Nodes in the Simulation Setup

The Ethernet network consists of several nodes, all of which are described in the database. For the simulation, you must add these in the Simulation Setup (Configuration desktop or Simulation ribbon tab|Setup).

Step by Step Procedure

  1. Select theVehicle_Ethernet database in the System View of the Simulation Setup.

  2. Open the Database Import Assistant dialog via the Node Synchronization... context menu command.

  3. Move the ADAS, CAMF, IC, VGW nodes from the Available nodes overview to the Assigned nodes overview using [>>].

  4. After selection, click [Next >].
    You will now see the added nodes in the network overview.

  5. Click [Finish].
    The nodes are now displayed in the Simulation Setup.

Add an Interaction Layer

Various aspects of an Ethernet node can be simulated. As a rule, not all of these are required or intended for a simulation. You can therefore determine the scope of the simulation by assigning the required interaction layers.

Select the network symbol Ethernet1.

Open the file selection via the context menu command Assign Module to All Network Nodes....

Select the following interaction layers one after the other:

  • C:\ProgramData\Vector\AddOn Packages\Vector AddOn\vModules\AsrPDUIL2.vmodule for the AUTOSAR PDU handling

  • <Installationsverzeichnis>\Exec64\CANoeILNL_AUTOSAR_Eth.vmodule for the SOME/IP services

Via the context menu command Configuration..., you can also assign interaction layers to each network node individually. In the open Node Configuration dialog, switch to the Components page and add the interaction layers there. You will now see the two files already assigned in the Node Configuration dialog.

Starting the Simulation

Step by Step Procedure

  1. Start the measurement with <F9> or with on the Home|Measurement|Start ribbon tab.

  2. Switch to the Trace desktop and pause the Trace Window with in the toolbar.

  3. Use the toolbar to select the Ethernet IP/Services column layout.

    The sequence originates from a packet sent by the simulated VGW node. It is a SOME/IP service discovery packet that is sent to all interested participants in the network.

Display of the Lines in the Trace Window

In contrast to the display of data traffic on a CAN bus, considerably more lines are displayed for Ethernet.

The Sim column is a very useful indicator. The letter s is displayed in the first two lines, but not in the others. The s means that the event was generated by the CANoe DE simulation. It is not simply forwarded by a switch or received by the real network.

The time stamps of the first two lines are identical. If the time stamps of two or more lines are exactly the same, it is very likely that it is the same event but interpreted differently. In this case, the event is first displayed as an Ethernet packet and then interpreted as a SOME/IP special event. Further information on the event can be found either via the [+] at the beginning of the line or via the Detail view ( in the toolbar).

But why are the other lines displayed? The destination is a multicast address that is directed to all SOME/IP nodes in the network. This means that we expect the packet to be received by all our simulation nodes. To get an even better understanding, we need to look at the simulated network.

The Ethernet packet is sent by the VGW. Switch2 receives it and sends it to the measuring port Port2. The VN interface sends it to the measurement port Port1 via the loopback cable. Switch1 forwards it to the simulation port IC. All this can be seen in the above excerpt from the Trace Window.

Signal Values Coming From the Simulation

The interaction layers read the information from the database in order to simulate the network nodes. For example, the SOME/IP service was announced in the example above and subscription requests are processed. However, the database does not contain any information about when the signals should be assigned a specific value. For this reason, the signal values still correspond to their start values.

Change the Signal Values via the Symbol Explorer

Step by Step Procedure

  1. Open the Symbol Explorer (Measurement desktop or Environment ribbon tab).

  2. Select the VehicleSpeed.signal.

  3. Open the Symbol Panel via the context menu command Change value....

    In the Symbol Panel, you can set the value via direct value entry or via a slider.

  4. Make sure that the Graphics Window or Data Window is visible on the Measurement desktop

  5. Start the measurement.

  6. CANoe DE places the new signal value in the designated position in the AUTOSAR PDU, which is transmitted periodically according to the database information.

  7. Now select the SignType signal in the Symbol Explorer; the quickest way to find it is to search for it in the Symbol Explorer.

  8. Drag and drop the signal into the Data Window.

  9. Change the value of the signal via the Symbol Panel.

    As a value table is stored for this signal, you can only change the values via the selection list in the Symbol Panel.

Create a Panel to Change the Signal Values

You can also change signal values using graphical panels.

You can create the panels with the Panel Designer, save them in a file and then add them to the current configuration.

Step by Step Procedure

  1. Open the Panel Designer via the Tools ribbon tab. This ensures that the database is available. A new panel is opened automatically.

  2. Select the Track Bar in the Toolbox on the right under Common and place it on the panel using drag-and-drop.

  3. Select the VehicleSpeed signal in the Symbol Explorer on the left and assign it to the Track Bar using drag-and-drop.

    In the Properties of the Track Bar on the right-hand side, you will now see the assigned signal under Symbol Values. You can also select a new signal here.

  4. Save the panel using the quick access toolbar.

    Create the Panles subdirectory in your configuration directory and save the panel under the name SIGNAL.

  5. In the Add to Project dialog that now opens, activate your panel in the first column and confirm your entries with [OK].

  6. Select the Input/Output Box in the Toolbox on the right under Common and place it on the panel using drag-and-drop.

  7. Select the EMotorCurrent signal in the Symbol Explorer on the left and assign it to the Input/Output Box using drag-and-drop.

  8. Select the Combo Box in the Toolbox on the right under Common and place it on the panel using drag-and-drop.

  9. Select the SignType signal in the Symbol Explorer on the left using the search input and assign it to the Input/Output Box using drag-and-drop.

  10. You can change the display text via the Properties on the right under General|Text.

Save your configuration and start the measurement. You can now change the signal values of these three signals with your new panel.
