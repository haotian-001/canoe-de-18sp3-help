Hardware Setup for 10BASE-T1S

10BASE-T1S is a multi-drop Ethernet network that differs fundamentally from the other automotive Ethernet standards. It is not fully switched. Instead, several participants can connect to the same bus line. The configuration is therefore different.

When 10BASE-T1S is operated in PLCA (Physical Layer Collision Avoidance) mode, each physical node is given an individual transmission opportunity to send Ethernet packets, while all other nodes are not authorized to do so. Collisions are then completely avoided. Each node is configured with a unique node ID and the transmission opportunity is passed from one node to the next according to the numerical order of their IDs.

Each measurement port of a VN interface connected to a 10BASE-T1S network is assigned a unique ID. If the ID is 0, the VN interface also sends the beacon to the network. The beacon is only sent by one node. It is the trigger for the next round of transmission options.

The following illustration shows the hardware setup of a VN5650 with two VNmodule60 1AE10LM Piggies for our simulation. Instead of the two normal switch segments, two Multidrop segments are used. The advantage of the Multidrop segments is that each simulation node can send an Ethernet packet for its measurement port at every transmission opportunity.

Note

A Multidrop segment is only available for 10BASE-T1S networks. It is like a switch, but there can be multiple simulation nodes behind a measurement port. This is a trick, because physically each node needs a unique node ID.
