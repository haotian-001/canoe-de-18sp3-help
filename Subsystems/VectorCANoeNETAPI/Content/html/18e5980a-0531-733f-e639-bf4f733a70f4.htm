Remarks
For the ARINC 429 messages defined in databases, specific classes are generated in type libraries. These specific classes are subclasses of A429Message and provide easy access to signal values. A429Message can also be used as a superclass for your own classes. Then, you can use the Signal attribute to define the layout of signals in the message. You can also create instances of A429Message directly. Then you should set the Label manually before sending the message. The payload can only be set on a per-byte basis in the A429Message class. You also receive objects of this class in the OnAnyA429Message handler.
