Remarks
Through the use of this attribute, subclasses of A429Message can define the layout of signals in the message. The field which is marked with this attribute must be a public field of the correct data type for the signal. The attribute receives in its constructor the start bit and bit count of the signal. Other properties are provided for the physical value calculation and the byte order. Whenever you read or write the signal value, not the field itself is read or written, but the data bytes of the message are changed. The read or written values are automatically assumed to be the physical values of the signals, and are therefore of type double. It is not possible to read or write the raw values directly.
