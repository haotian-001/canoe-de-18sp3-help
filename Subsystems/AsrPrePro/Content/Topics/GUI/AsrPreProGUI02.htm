Apply Custom Patches

Use customized patches

Activate this check box to use customer-specific patches.

Customer-specific patches enable the loading and execution of specific operations that are applied to the merged ARXML files at AUTOSAR Preprocessor runtime. They are provided in the form of dynamic program libraries (Dynamic Link Libraries, DLLs) in the installation directory of AUTOSAR Preprocessor and consist of an assembly name and a patch configuration file (configuration). (See custom-patches).

The patch configuration file is specified in YAML format and is used to parameterize the patch. The format, type, number and function of the parameters are determined by the custom patch and therefore vary from patch to patch. Corresponding template configuration files are stored under C:\ProgramData\Vector\AUTOSAR Preprocessor.

The order of execution depends on the priority. The patch with priority #1 is executed first.

Adding Custom Patch

You can add a customized patch by clicking on this button. A list entry consists of a patch file and a configuration file.

  • Patch File <dll>
  • You can use this input box to set the subparameter assembly-name of custom-patch. This subparameter defines the patch file, that is to be executed and must be selected for each list element.
  • You can add and select patches via a dialog using the More button in the drop-down list of the patch file input field.
  • Configuration <yaml>
  • You can use this input field to set the subparameter configuration of custom-patch. This subparameter defines the configuration file of the respective patch.
  • You can use the More button in the drop-down list of the configuration input box to add and select patch configuration files via a dialog.

Remove Custom Patch

Click on this button to remove the corresponding cutomer-specific patch.

Increase Patch Priority

By clicking on this button, you can increase the execution priority of the corresponding patch.

Reduce Patch Priority

By clicking on this button, you can reduce the execution priority of the corresponding patch.
